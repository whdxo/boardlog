## 결정 사항

1. **백엔드** — MVP는 Next.js API Routes 사용. 트래픽 증가 시 별도 서버 분리 검토
2. **구매/결제** — MVP는 외부 쇼핑몰 링크 연결 (네이버 쇼핑 등). Phase 2에서 자체 결제 (토스페이먼츠 등) 검토
3. **초기 보드게임 데이터** — BGG XML API2 (`boardgamegeek.com/xmlapi2/`) 활용
   - `/hot?type=boardgame` — 인기 게임 50개로 시작
   - `/thing?id=1,2,3` — 상세 데이터 일괄 수집 (최대 ~20개씩)
   - `/search?query=&type=boardgame` — 검색
   - 제한: 초당 1~2 요청, API 키 불필요, XML 포맷
   - BGG에 없는 국내 게임은 직접 입력

## 운영 정책 (MVP 기준)

### 이미지 업로드
- 게시글: 최대 5장, 장당 5MB 이하
- 프로필: 1장, 2MB 이하
- 허용 포맷: JPG, PNG, WebP
- 서버에서 리사이즈 (최대 1200px)

### 플레이 로그 수정·삭제
- 작성자 본인만 수정·삭제 가능
- 삭제 시 soft delete (DB에서 실제 삭제하지 않음)

### 구매 링크 관리
- 게임당 여러 쇼핑몰 링크 등록 가능
- 가격 정보 수동 등록 (Phase 2에서 자동 크롤링 검토)
- 품절/링크 만료 시 비활성 처리
