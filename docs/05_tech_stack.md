## 프론트엔드


| 항목      | 선택                   | 이유                     |
| ------- | -------------------- | ---------------------- |
| 프레임워크   | Next.js (App Router) | SEO 필요 (게임 DB), 풀스택 가능 |
| 언어      | TypeScript           | 안정성                    |
| 스타일     | Tailwind CSS         | 빠른 개발                  |
| 상태관리    | Zustand or Jotai     | 가볍고 단순                 |
| 서버 상태   | TanStack Query       | 캐싱, 비동기 처리             |
| UI 컴포넌트 | shadcn/ui            | Tailwind 기반, 커스텀 쉬움    |

## 결정 필요 사항 (논의)

1. **백엔드 분리 여부** — Next.js API Routes만 쓸 것인지, NestJS 등 별도 서버를 둘 것인지
2. **펀딩 결제** — 자체 결제(토스페이먼츠 등) vs 외부 링크 연결만
3. **초기 보드게임 데이터** — BGG 크롤링 vs 직접 입력 vs API
