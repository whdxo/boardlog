## 프롬프트 1 — 프로젝트 세팅 + 디자인 시스템

```
보드게임 종합 플랫폼 "BoardLog"의 프론트엔드를 만들어줘.

기술 스택:
- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui 컴포넌트
- Lucide Icons
- 폰트: Pretendard (한국어), Inter (영문/숫자)

디자인 시스템:
- Primary: #3B82F6 (파란색), hover #2563EB, light #DBEAFE
- Accent: #FBBF24 (별점 노란색)
- Success: #10B981, Error: #EF4444
- Gray: 900 #111827, 600 #4B5563, 400 #9CA3AF, 200 #E5E7EB, 50 #F9FAFB
- 배경: #F9FAFB (페이지), #FFFFFF (카드)
- border-radius: 카드 12px, 버튼/인풋 8px, 태그 9999px (pill)
- shadow-sm: 0 1px 2px rgba(0,0,0,0.05), shadow-md: 0 4px 6px rgba(0,0,0,0.07)

타이포:
- display 28px Bold, h1 22px Bold, h2 18px SemiBold, h3 16px SemiBold
- body 14px Regular, caption 12px Regular

간격: 4px 단위 (4, 8, 12, 16, 24, 32, 48)

반응형:
- Mobile: ~767px (1열)
- Tablet: 768~1023px (2열)
- Desktop: 1024px~ (max-width 1080px, 중앙 정렬, 양쪽 여백 넉넉하게)

다크모드는 지원하지 않음. 라이트 모드만.

공통 컴포넌트를 먼저 만들어줘:
1. Button (Primary, Secondary, Ghost, Danger / sm, md, lg)
2. Input (기본, 포커스, 에러, 비활성)
3. GameCard (썸네일 3:4, 게임명, 평점 ★, 가격, hover시 translateY -2px)
4. Tag/Badge (pill 형태, 다양한 색상 variant)
5. StarRating (5점 만점, 1점 단위, accent 노란색, 선택 시 바운스)
6. Toast (성공 초록, 에러 빨강, 정보 파랑, 하단 표시 3초)
7. Modal (데스크톱: 중앙 max 480px / 모바일: 바텀시트 하단에서 슬라이드업)
8. Skeleton (카드형, 리스트형, gray-100 펄스 애니메이션)
9. EmptyState (아이콘 + 메시지 + CTA 버튼)
```

## 프롬프트 2 — 공통 레이아웃

```
BoardLog의 공통 레이아웃을 만들어줘. 모든 페이지가 이 레이아웃 안에 들어감.

모바일 레이아웃 (~767px):
- 상단 헤더 56px 고정
  - 메인 페이지: 좌측 "BoardLog" 텍스트 로고 (18px Bold, #3B82F6) + 우측 알림🔔, 설정⚙️ 아이콘
  - 상세 페이지: 좌측 ← 뒤로가기 + 중앙 페이지 제목 + 우측 더보기⋮
  - 배경 white, 하단 보더 #E5E7EB 1px
- 콘텐츠 영역: flex-1, 스크롤 가능
- 하단 탭바 56px 고정 + iOS safe area
  - 4탭: 홈(Home), 탐색(Dice5), 내기록(BookOpen), 프로필(User)
  - 활성: filled 아이콘 + #3B82F6 텍스트
  - 비활성: outline 아이콘 + #9CA3AF 텍스트
  - 아이콘 24px, 텍스트 11px, 배경 white, 상단 보더

데스크톱 레이아웃 (1024px~):
- 상단 GNB 64px 고정
  - 좌측: "BoardLog" 로고 (22px Bold, #3B82F6)
  - 중앙: 홈, 탐색, 내기록 메뉴 (14px, 간격 32px, 활성시 #3B82F6 + 하단 2px 바)
  - 우측: 알림 아이콘 + 프로필 (32px 원형 아바타 + 닉네임 + 드롭다운)
  - 비로그인: [로그인] Ghost 버튼 + [회원가입] Primary 버튼
  - 배경 white, 하단 shadow-sm
  - 프로필 드롭다운: 내 프로필, 설정, 구분선, 로그아웃 (180px, radius 8px, shadow-md)
- 콘텐츠: max-width 1080px, 중앙 정렬, 양쪽 여백 자동
- 하단 푸터
  - 배경 #F9FAFB, 상단 보더
  - 좌측: BoardLog 로고 + 설명
  - 우측: 서비스(게임 탐색, 내 기록), 지원(이용약관, 개인정보, 가이드라인)
  - 하단: © 2026 BoardLog

탭바/GNB는 항상 고정. 하단 탭바는 모바일에서만, 푸터는 데스크톱에서만.

비로그인 시: 내기록/프로필 탭 클릭하면 로그인 유도 화면 (🔒 아이콘 + "로그인이 필요한 기능이에요" + 로그인/회원가입 버튼)
```

## 프롬프트 3 — 홈 페이지

```
BoardLog 홈 페이지를 만들어줘. 허브 역할로 모든 섹션의 요약을 보여줌.

섹션 순서 (위→아래):

1. 검색바
- 높이 44px, 배경 #F9FAFB, 보더 #E5E7EB, radius 8px
- 🔍 아이콘 + "보드게임을 검색해보세요" 플레이스홀더
- 탭하면 게임 목록 페이지(/games)로 이동 (홈에서는 입력 불가, 트리거 역할)

2. 배너 슬라이드
- 비율: 모바일 16:9, 데스크톱 3:1
- radius 12px, 좌우 마진 16px (모바일)
- 하단 인디케이터 점, 활성 #3B82F6 / 비활성 #D1D5DB
- 5초 자동 전환, 스와이프 가능, 최대 5장
- 배너 위 텍스트: 하단 그라데이션 오버레이 + 흰 텍스트
- 더미 배너 3장 (신작 게임, 할인 이벤트, 추천 등)

3. 🔥 인기 게임
- 섹션 제목 22px Bold + "전체 보기 >" 우측 (12px, #4B5563)
- 모바일: 가로 스크롤, 카드 너비 140px, 간격 12px, 스냅
- 데스크톱: 4열 그리드, gap 24px
- GameCard 컴포넌트 사용 (썸네일 + 게임명 + ★평점 + 가격)
- 가격: 14px SemiBold, #3B82F6
- 더미 데이터 8개

4. 📚 내 최근 기록 (로그인 시)
- 섹션 제목 + "전체 보기 >"
- 최근 플레이 로그 3개 리스트
  - 🎲 게임명 · 날짜 (우측)
  - 👥 함께한 사람  📍 장소
- 하단 "[ + 새 기록 작성하기 ]" Secondary 버튼 전체 너비
- 비로그인 시: 카드 형태 로그인 유도 (🎲 아이콘 + "플레이 기록을 시작해보세요" + 로그인/회원가입 버튼)

5. 🏷️ 추천 게임
- 인기 게임과 동일 레이아웃
- 더미 데이터 8개 (다른 게임)

데스크톱에서 "내 최근 기록"과 "추천 게임"은 2컬럼 나란히 배치.
모든 데이터는 더미 데이터로 채워줘 (한국어 보드게임 이름 사용).
```

## 프롬프트 4 — 탐색 (게임 목록 + 게임 상세)

```
BoardLog 탐색 페이지 2개를 만들어줘.

[1] 게임 목록 (/games)

상단:
- 검색바: 높이 44px, "게임명, 장르, 디자이너 검색", 입력 시 실시간 필터링 (300ms 디바운스)
- 필터 칩: [인원수 ▾] [가격대 ▾] [필터 더보기 ▾]
  - 기본 칩 스타일: #F3F4F6 배경, #374151 텍스트, pill, 32px 높이
  - 선택 시: #DBEAFE 배경, #2563EB 텍스트
  - 인원수 옵션: 1인, 2인, 3~4인, 5인 이상
  - 가격대 옵션: ~2만원, 2~4만원, 4~6만원, 6만원~
  - "필터 더보기" 탭 시 바텀시트(모바일) / 드롭다운(데스크톱): 장르(전략,가족,파티,추리,협력,경제,카드 - 다중선택), 플레이시간(~30분,30~60분,60~120분,2시간~), 연령(전체,8+,12+,14+)
  - 적용 버튼에 결과 수 실시간 표시
- 정렬: "인기순 ▾" 드롭다운 (인기순, 평점 높은순, 가격 낮은순, 가격 높은순, 최신순) + "128개 결과"

게임 그리드:
- 모바일 2열 (gap 12px), 데스크톱 4열 (gap 24px)
- GameCard 컴포넌트 재사용
- 무한 스크롤
- 결과 없을 때: 🔍 + "검색 결과가 없어요" + 안내 텍스트
- 더미 데이터 20개 이상 (한국어 보드게임)

[2] 게임 상세 (/games/:id)

모바일:
- 게임 이미지 (전체 너비, 1:1)
- 게임명 (28px Bold) + 영문명 (12px, #9CA3AF)
- ★★★★☆ 4.2 (128명 평가) — 별 #FBBF24
- 정보 칩: [👥 2~4인] [⏱ 60~90분] [🔞 10세+] — pill, #F3F4F6 배경
- [☆ 내 평점 등록하기] Secondary 버튼 → 바텀시트(별 5개 40px, 저장 버튼)
- [+ 컬렉션에 추가] Ghost 버튼 → 바텀시트(보유/위시리스트/플레이완료 선택)
- 게임 소개: 3줄 말줄임 + [더보기] 토글
- 상세 정보: 디자이너, 퍼블리셔, 출시년도, 장르 (키-값 리스트)
- 평점 분포: 5~1점 가로 바 차트 (#FBBF24 채움, #F3F4F6 빈 부분)
- 연관 게임: 가로 스크롤 GameCard
- 하단 고정 구매바: 높이 64px, white 배경, 상단 보더
  - 좌측: 가격 (18px Bold)
  - 우측: [구매하기 →] Primary 버튼
  - 탭하면 외부 링크 새 탭으로 열림

데스크톱:
- 좌측: 게임 이미지 400×400, radius 12px
- 우측: 게임 정보 + 하단에 구매 카드 (sticky)
  - 구매 카드: 280px, radius 12px, 보더, shadow-sm
  - 가격 22px Bold + [구매하기 →] Primary 버튼 전체 너비 + "네이버 쇼핑에서 구매" caption
- 아래: 소개, 평점 분포, 연관 게임 (4열 그리드)

더미 데이터로 상세 페이지 1개 완성해줘 (카탄으로).
```

## 프롬프트 5 — 내 기록

```
BoardLog 내 기록 섹션 페이지들을 만들어줘.

[1] 내 기록 메인 (/my)

상단 통계 카드 3개 (3열 균등):
- 🎲 12 보유 / 📝 8 플레이 / ⭐ 15 평점
- 배경 #F9FAFB, radius 12px, 높이 72px
- 숫자 22px Bold, 라벨 12px #4B5563
- 탭하면 해당 탭으로 전환

탭 바: [컬렉션] [로그] [평점] [북마크]
- 4등분, 활성 탭 #3B82F6 + 하단 2px 바, 비활성 #4B5563
- sticky (스크롤 시 상단 고정)
- 탭 전환 시 하단 바 슬라이드 애니메이션

[2] 컬렉션 탭
- 서브 필터: [전체 12] [보유 8] [위시 3] [완료 1] — pill 칩
- 리스트: 좌측 썸네일(60×80, 3:4) + 게임명(16px SemiBold) + ★평점·인원 + 가격 + 상태 뱃지
  - 보유: #10B981 배경, 위시: #FEF3C7 배경, 완료: #F3F4F6 배경
- 빈 상태: 🎲 + "아직 컬렉션이 비어있어요" + [게임 탐색하기] 버튼
- 더미 8개

[3] 로그 탭
- 월별 그룹핑: "2026년 3월" 헤더 (18px SemiBold, sticky)
- 로그 카드: 날짜 + 🎲게임명 + 👥함께한 사람 + 📍장소 + ★별점
  - white 배경, radius 12px, shadow-sm, 간격 12px
- 우하단 FAB: 56px 원형, #3B82F6, 흰 + 아이콘, shadow-md
  - 데스크톱에서는 FAB 대신 상단 우측 [+ 기록하기] Primary 버튼
- 빈 상태: 📝 + "아직 플레이 기록이 없어요" + [+ 기록하기]
- 더미 5개 (2개월 분)

[4] 플레이 로그 작성 (/my/logs/write)
- 폼 필드:
  - 게임 선택 *: 검색 인풋 → 탭 시 게임 검색 모달 (바텀시트, 검색바 + 게임 리스트)
  - 플레이 날짜 *: 날짜 피커, 기본값 오늘, 미래 불가
  - 함께한 사람: 텍스트 입력 + 엔터 → pill 칩 추가 (✕ 삭제)
  - 장소: 텍스트 인풋, "어디서 했나요?"
  - 별점: ☆ 5개 (32px), 선택 사항
  - 메모: textarea 4줄, "오늘 게임 한 줄 감상..."
- 모바일: 전체 화면, 헤더에 [저장] 버튼
- 데스크톱: 중앙 카드 max 600px, 하단 [저장하기] Primary 버튼
- 게임 + 날짜만 필수, 나머지 선택

[5] 플레이 로그 상세 (/my/logs/:id)
- 게임명(22px Bold, 탭 시 게임 상세로) + 별점 + 날짜 + 사람 + 장소 + 메모
- 헤더 우측 ⋮ → 수정/삭제 드롭다운
- 삭제 시 확인 모달: "이 기록을 삭제할까요?" + [취소] [삭제하기 (빨강)]

[6] 평점 탭
- 리스트: 컬렉션과 동일 레이아웃 + 우측에 ★★★★☆ 표시
- 빈 상태: ⭐ + "아직 평가한 게임이 없어요"
- 더미 5개

[7] 북마크 탭
- 리스트: 컬렉션과 동일 + 우측 🔖 아이콘 (filled, #3B82F6)
- 빈 상태: 🔖 + "북마크한 게임이 없어요"
- 더미 3개

모든 탭에 빈 상태 컴포넌트 포함. 더미 데이터 한국어.
```

## 프롬프트 6 — 계정 (로그인, 회원가입, 프로필, 설정)

```
BoardLog 계정 관련 페이지들을 만들어줘.

[1] 로그인 (/login)
- 상단: "BoardLog" 로고 (28px Bold, #3B82F6) + "보드게임을 발견하고 기록하세요" (12px, #9CA3AF)
- 소셜 로그인 버튼 3개 (높이 48px, radius 8px, 간격 8px):
  1. 카카오: 배경 #FEE500, 텍스트 #191919
  2. Apple: 배경 #000000, 텍스트 white
  3. Google: 배경 white, 보더 #E5E7EB, 텍스트 #191919
- "── 또는 ──" 구분선
- 이메일 인풋 + 비밀번호 인풋 (👁 보기 토글)
- [로그인] Primary 버튼 전체 너비
- "비밀번호를 잊으셨나요?" 링크 중앙
- 하단: "계정이 없으신가요? 회원가입" 링크
- 데스크톱: 중앙 카드 max 400px

[2] 회원가입 (/signup)
- 소셜 로그인 3개 (로그인과 동일)
- "── 또는 ──"
- 폼: 이메일*, 비밀번호*(8자 이상), 비밀번호 확인*, 닉네임*(2~12자)
- 닉네임: 500ms 디바운스 중복 체크 (✓ 초록 / ✕ 빨강 인라인)
- ☑ 이용약관 및 개인정보처리방침 동의 체크박스
- [가입하기] Primary 버튼
```

## 사용 팁

1. **반드시 순서대로** — 1번에서 만든 컴포넌트를 2~6번에서 재사용
2. **한 프롬프트당 하나씩** — 결과 확인 후 다음으로 넘어가기
3. **수정이 필요하면** — "게임 카드 가격 폰트를 16px로 키워줘" 같이 구체적으로
4. **더미 데이터** — 한국 보드게임 이름 사용 (카탄, 스플렌더, 윙스팬, 아줄, 코드네임, 티켓투라이드, 루트, 팬데믹 등)
5. **페이지 연결** — 각 프롬프트 완료 후 "게임 카드 클릭하면 /games/:id로 이동하게 연결해줘" 추가 요청
