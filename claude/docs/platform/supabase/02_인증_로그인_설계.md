# 인증/로그인 설계 — Supabase Auth

---

## 1. 인증 전략 개요

| 항목 | 선택 |
|------|------|
| 인증 제공자 | Supabase Auth (GoTrue) |
| 로그인 방식 | 카카오 소셜 로그인 (1순위), 구글 (2순위), 이메일/비밀번호 (3순위) |
| 세션 관리 | Supabase Auth + HTTP-only 쿠키 (`@supabase/ssr`) |
| 토큰 | JWT (access token + refresh token, 자동 갱신) |
| 권한 제어 | RLS (Row Level Security) |

### 왜 Supabase Auth인가?
- DB(PostgreSQL)와 동일 프로젝트에서 관리 → RLS와 자연스럽게 연동
- 카카오/구글 OAuth 내장 지원
- 별도 인증 서버 불필요
- `@supabase/ssr`로 Next.js App Router와 완벽 호환

## 2. 소셜 로그인 설정

### 2-1. 카카오 로그인

#### Kakao Developers 설정
1. [developers.kakao.com](https://developers.kakao.com) 접속
2. 애플리케이션 추가 → `BoardLog`
3. **플랫폼 등록**:
   - 웹: `http://localhost:3000` (개발), `https://boardlog.app` (운영)
4. **카카오 로그인 활성화**:
   - 카카오 로그인 → 활성화 ON
   - Redirect URI: `https://<project-ref>.supabase.co/auth/v1/callback`
5. **동의항목 설정**:
   - 닉네임: 필수
   - 프로필 사진: 선택
   - 이메일: 선택 (카카오 계정 이메일)
6. **키 확인**:
   - REST API 키 → Supabase에 입력
   - Client Secret → Supabase에 입력

#### Supabase Dashboard 설정
- Authentication → Providers → Kakao
  - **Client ID**: REST API 키
  - **Client Secret**: Client Secret 키
  - Enabled: ON

### 2-2. 구글 로그인

#### Google Cloud Console 설정
1. [console.cloud.google.com](https://console.cloud.google.com) 접속
2. 프로젝트 생성 → `BoardLog`
3. APIs & Services → OAuth consent screen 설정
4. Credentials → OAuth 2.0 Client ID 생성
   - Application type: Web application
   - Authorized redirect URI: `https://<project-ref>.supabase.co/auth/v1/callback`
5. Client ID, Client Secret 확인

#### Supabase Dashboard 설정
- Authentication → Providers → Google
  - **Client ID**: OAuth Client ID
  - **Client Secret**: OAuth Client Secret
  - Enabled: ON

### 2-3. 이메일/비밀번호

Supabase Auth에서 기본 제공. 별도 설정 불필요.

#### Supabase Dashboard 설정
- Authentication → Providers → Email
  - Confirm email: ON (이메일 인증 필수)
  - Secure email change: ON

## 3. 인증 플로우

### 3-1. 소셜 로그인 플로우 (카카오/구글)

```
[사용자] → "카카오로 로그인" 클릭
    ↓
[클라이언트] supabase.auth.signInWithOAuth({ provider: 'kakao' })
    ↓
[브라우저] 카카오 로그인 페이지로 리다이렉트
    ↓
[카카오] 사용자 인증 → Supabase callback URL로 리다이렉트
    ↓
[Supabase Auth] auth.users에 유저 생성/업데이트 + JWT 발급
    ↓
[트리거] on_auth_user_created → profiles 테이블에 행 자동 생성
    ↓
[미들웨어] 쿠키에 세션 저장
    ↓
[클라이언트] / (홈)으로 리다이렉트
```

### 3-2. 이메일 회원가입 플로우

```
[사용자] 이메일, 비밀번호, 닉네임 입력
    ↓
[클라이언트] supabase.auth.signUp({ email, password, options: { data: { nickname } } })
    ↓
[Supabase Auth] auth.users에 유저 생성 + 확인 메일 발송
    ↓
[트리거] on_auth_user_created → profiles 생성 (닉네임 포함)
    ↓
[사용자] 이메일 확인 링크 클릭
    ↓
[Supabase Auth] 이메일 확인 완료 → 세션 활성화
```

### 3-3. 로그아웃 플로우

```
[사용자] 로그아웃 클릭
    ↓
[클라이언트] supabase.auth.signOut()
    ↓
[Supabase Auth] 세션 파기 + 쿠키 삭제
    ↓
[클라이언트] /login으로 리다이렉트
```

## 4. 구현 코드

### 4-1. 라우트 구조

```
src/app/
  (auth)/
    login/page.tsx          ← 로그인 페이지
    signup/page.tsx          ← 회원가입 페이지
    auth/callback/route.ts   ← OAuth 콜백 핸들러
    auth/confirm/route.ts    ← 이메일 확인 핸들러
```

### 4-2. OAuth 콜백 핸들러 (`auth/callback/route.ts`)

```typescript
import { NextResponse } from "next/server";
import { createClient } from "@/lib/supabase/server";

export async function GET(request: Request) {
  const { searchParams, origin } = new URL(request.url);
  const code = searchParams.get("code");
  const next = searchParams.get("next") ?? "/";

  if (code) {
    const supabase = await createClient();
    const { error } = await supabase.auth.exchangeCodeForSession(code);

    if (!error) {
      return NextResponse.redirect(`${origin}${next}`);
    }
  }

  // 에러 시 로그인 페이지로
  return NextResponse.redirect(`${origin}/login?error=auth_failed`);
}
```

### 4-3. 로그인 페이지 핵심 로직

```typescript
"use client";

import { createClient } from "@/lib/supabase/client";

export default function LoginPage() {
  const supabase = createClient();

  // 카카오 로그인
  const handleKakaoLogin = async () => {
    await supabase.auth.signInWithOAuth({
      provider: "kakao",
      options: {
        redirectTo: `${window.location.origin}/auth/callback`,
      },
    });
  };

  // 구글 로그인
  const handleGoogleLogin = async () => {
    await supabase.auth.signInWithOAuth({
      provider: "google",
      options: {
        redirectTo: `${window.location.origin}/auth/callback`,
        queryParams: {
          access_type: "offline",
          prompt: "consent",
        },
      },
    });
  };

  // 이메일 로그인
  const handleEmailLogin = async (email: string, password: string) => {
    const { error } = await supabase.auth.signInWithPassword({
      email,
      password,
    });

    if (error) {
      // 에러 처리
      return;
    }

    window.location.href = "/";
  };

  // 회원가입
  const handleSignUp = async (email: string, password: string, nickname: string) => {
    const { error } = await supabase.auth.signUp({
      email,
      password,
      options: {
        data: { nickname },
      },
    });

    if (error) {
      // 에러 처리
      return;
    }

    // 이메일 확인 안내
  };

  // ... UI 렌더링
}
```

### 4-4. 서버에서 유저 확인 (Server Component)

```typescript
import { createClient } from "@/lib/supabase/server";

export default async function MyPage() {
  const supabase = await createClient();
  const { data: { user } } = await supabase.auth.getUser();

  if (!user) {
    // 비로그인 → 로그인 유도
    return <LoginPrompt />;
  }

  // 프로필 조회
  const { data: profile } = await supabase
    .from("profiles")
    .select("*")
    .eq("id", user.id)
    .single();

  return <MyPageContent profile={profile} />;
}
```

### 4-5. 클라이언트에서 인증 상태 관리 (Zustand 연동)

```typescript
// src/stores/authStore.ts (개선안)
import { create } from "zustand";
import { createClient } from "@/lib/supabase/client";
import type { User } from "@supabase/supabase-js";

interface AuthState {
  user: User | null;
  profile: Profile | null;
  isLoading: boolean;
  initialize: () => Promise<void>;
  signOut: () => Promise<void>;
}

export const useAuthStore = create<AuthState>((set) => ({
  user: null,
  profile: null,
  isLoading: true,

  initialize: async () => {
    const supabase = createClient();
    const { data: { user } } = await supabase.auth.getUser();

    if (user) {
      const { data: profile } = await supabase
        .from("profiles")
        .select("*")
        .eq("id", user.id)
        .single();

      set({ user, profile, isLoading: false });
    } else {
      set({ user: null, profile: null, isLoading: false });
    }

    // 인증 상태 변경 구독
    supabase.auth.onAuthStateChange(async (event, session) => {
      if (session?.user) {
        const { data: profile } = await supabase
          .from("profiles")
          .select("*")
          .eq("id", session.user.id)
          .single();

        set({ user: session.user, profile });
      } else {
        set({ user: null, profile: null });
      }
    });
  },

  signOut: async () => {
    const supabase = createClient();
    await supabase.auth.signOut();
    set({ user: null, profile: null });
  },
}));
```

## 5. 보호된 라우트

### 5-1. 미들웨어에서 인증 체크

```typescript
// src/middleware.ts (확장)
import { type NextRequest, NextResponse } from "next/server";
import { updateSession } from "@/lib/supabase/middleware";
import { createServerClient } from "@supabase/ssr";

// 로그인 필수 경로
const PROTECTED_PATHS = [
  "/my",
  "/community/new",
  "/used/write",
  "/settings",
  "/profile/edit",
];

export async function middleware(request: NextRequest) {
  const response = await updateSession(request);
  const { pathname } = request.nextUrl;

  // 보호된 경로 체크
  const isProtected = PROTECTED_PATHS.some((path) => pathname.startsWith(path));

  if (isProtected) {
    const supabase = createServerClient(
      process.env.NEXT_PUBLIC_SUPABASE_URL!,
      process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
      {
        cookies: {
          getAll() {
            return request.cookies.getAll();
          },
          setAll() {},
        },
      }
    );

    const { data: { user } } = await supabase.auth.getUser();

    if (!user) {
      const loginUrl = new URL("/login", request.url);
      loginUrl.searchParams.set("next", pathname);
      return NextResponse.redirect(loginUrl);
    }
  }

  return response;
}
```

### 5-2. 로그인 필수 vs 선택 페이지

| 구분 | 경로 | 비로그인 시 |
|------|------|-----------|
| **공개** | `/`, `/games`, `/community`, `/local`, `/shop`, `/used` | 정상 접근 |
| **공개 (일부 기능 제한)** | `/games/[id]` | 평가/컬렉션 버튼 → 로그인 시트 |
| **로그인 필수** | `/my/*`, `/community/new`, `/used/write`, `/settings/*` | 로그인 페이지 리다이렉트 |
| **비로그인 전용** | `/login`, `/signup` | 정상 접근 (로그인 시 홈으로) |

## 8. 보안 체크리스트

- [ ] `SUPABASE_SERVICE_ROLE_KEY`는 절대 클라이언트에 노출하지 않음
- [ ] `NEXT_PUBLIC_` 접두사가 없는 환경변수는 서버에서만 접근 가능 확인
- [ ] RLS 정책이 모든 테이블에 적용되어 있는지 확인
- [ ] OAuth redirect URI가 정확히 설정되어 있는지 확인
- [ ] CSRF 보호 (Supabase Auth 내장)
- [ ] 비밀번호 최소 요구사항 설정 (8자 이상)
- [ ] Rate limiting (Supabase Auth 내장, 추가 설정 가능)
