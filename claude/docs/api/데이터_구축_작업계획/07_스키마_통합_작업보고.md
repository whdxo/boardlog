# 스키마 통합 작업 보고

> 작성일: 2026-04-10  
> 작업: Prisma `board_games` → Supabase `games` 테이블 통합 + 앱 코드 컬럼명 일치

---

## 1. 작업 배경

### 문제

Tier 1 시딩 후 두 개의 독립적인 구조가 충돌하고 있었음:

| 구조 | 테이블명 | 컬럼 예시 |
|------|---------|----------|
| Prisma 스키마 (기존) | `board_games` | `thumbnail_url`, `year_published`, `play_time_min`, `bgg_rank` |
| 앱 코드 기대 (`games/page.tsx` 등) | `games` | `thumbnail`, `release_year`, `min_play_time`, `rank` |

앱 코드가 Supabase를 직접 쿼리하므로 테이블명과 컬럼명이 정확히 일치해야 함.

### 선택지 검토

| 옵션 | 장단점 |
|------|--------|
| A. Prisma 스키마를 앱 코드 기준으로 맞춤 | 깔끔, 재시딩 ~5분 |
| B. `games` VIEW 생성 (board_games 매핑) | collections FK 참조 불가, 임시방편 |

→ **A 선택** (재시딩이 CSV에서 바로 가능하므로 비용 낮음)

## 2. 변경 내용

### 2-1. 테이블명 변경

| 이전 | 이후 |
|------|------|
| `board_games` | `games` |
| `board_game_tags` | `game_tags` |
| `tags` | `tags` (유지) |

### 2-2. `games` 컬럼명 변경

| 이전 (`board_games`) | 이후 (`games`) | 앱 코드 필드명 |
|---------------------|---------------|--------------|
| `thumbnail_url` | `thumbnail` | `thumbnail` |
| `year_published` | `release_year` | `releaseYear` |
| `play_time_min` | `min_play_time` | `minPlayTime` |
| `play_time_max` | `max_play_time` | `maxPlayTime` |
| `bgg_rank` | `rank` | `rank` |
| `avg_rating` | `avg_rating` | `avgRating` (유지) |
| `bgg_id` (Int) | `bgg_id` (Text) | `bggId` |
| — (없었음) | `title_en` | `titleEn` |
| — (없었음) | `genres TEXT[]` | `genres` |
| — (없었음) | `rank_change` | `rankChange` |
| — (없었음) | `is_new` | `isNew` |

### 2-3. `bggId` 타입 변경

- 이전: `Int` (BGG ID를 숫자로 저장)
- 이후: `Text` (앱 코드가 문자열로 사용)

### 2-4. `genres` 필드 추가

- 이전: 태그 테이블만으로 장르 관리 (`board_game_tags`)
- 이후: `games.genres TEXT[]` 배열 + `game_tags` 테이블 **동시 유지**
  - `genres[]`: 앱 필터/표시용 (빠른 쿼리, JOIN 불필요)
  - `game_tags`: Tier 2 API 보강 후 세부 카테고리/메카닉 태그용

### 2-5. 신규 추가 테이블 (앱 코드 요구사항)

기존에는 `games`, `tags`, `game_tags` 3개만 있었음.  
앱 코드(`collection.ts`, `gameStats.ts`)가 쿼리하는 나머지 테이블을 Supabase에 생성:

| 테이블 | 용도 |
|--------|------|
| `profiles` | 유저 프로필 (auth.users 1:1 확장) |
| `collections` | 유저 게임 컬렉션 (8종 상태) |
| `ratings` | 유저 게임 평점 |
| `play_logs` | 플레이 기록 |
| `play_log_scores` | 플레이별 점수/승자 |

## 4. games 테이블 컬럼 전체 현황

```
id             UUID        PK, gen_random_uuid()
bgg_id         TEXT        UNIQUE (BGG 고유 ID)
title          TEXT        영문 게임명 (CSV name 컬럼)
title_kr       TEXT        한국어 게임명 (Tier 2 API에서 채울 예정)
title_en       TEXT        영문명 복사본 (Tier 1에서 title과 동일값으로 채움)
description    TEXT        게임 설명 (Tier 2 API에서 채울 예정)
min_players    INT         최소 인원 (Tier 2에서)
max_players    INT         최대 인원 (Tier 2에서)
min_play_time  INT         최소 플레이 시간 분 (Tier 2에서)
max_play_time  INT         최대 플레이 시간 분 (Tier 2에서)
min_age        INT         최소 권장 연령 (Tier 2에서)
genres         TEXT[]      장르 배열 (Tier 1: CSV rank 컬럼 기반 8종)
designer       TEXT        디자이너 (Tier 2에서)
publisher      TEXT        퍼블리셔 (Tier 2에서)
release_year   INT         출시 연도 (CSV yearpublished)
thumbnail      TEXT        썸네일 URL (Tier 2에서)
image_url      TEXT        고해상도 이미지 URL (Tier 2에서)
price          INT         가격 원화 (Tier 3 수동입력)
avg_rating     DECIMAL(3,1) 평균 평점 5점 만점 (CSV average ÷ 2)
rating_count   INT         평가 유저 수 (CSV usersrated)
rank           INT         BGG 전체 순위 (CSV rank)
rank_change    INT         순위 변화 (Tier 4 cron에서)
weight         DECIMAL(3,2) 난이도 1~5 (Tier 2에서)
is_new         BOOLEAN     신규 게임 여부 (Tier 4 cron에서)
is_expansion   BOOLEAN     확장팩 여부 (모두 false, 확장팩 제외 시딩)
last_synced_at TIMESTAMPTZ BGG API 마지막 동기화 시각
created_at     TIMESTAMPTZ
updated_at     TIMESTAMPTZ (트리거로 자동 갱신)
```

## 6. Prisma 스키마 변경 (prisma/schema.prisma)

```prisma
model Game {
  // @@map("games") — 테이블명
  // 주요 변경: bggId String (기존 Int), genres String[]
  // 컬럼명: releaseYear, rank, thumbnail 등 앱 코드 기준
}

model GameTag {
  // @@map("game_tags") — 기존 board_game_tags에서 변경
}
```

Prisma는 게임 데이터 시딩 전용으로만 사용.  
앱 런타임에서는 Supabase 클라이언트 직접 쿼리 (`src/lib/supabase/client.ts`).
