# 탐색 페이지 DB 연동 작업 계획

> 작성일: 2026-04-11  
> 상태: 작업 예정

---

## 1. 작업 배경

Tier 1 CSV 시딩으로 137,204개 게임이 DB에 들어가 있으나,
탐색 페이지(`/games`)에서 장르·순위·평가수 표시와 필터 동작이 구현되지 않아
실제 데이터가 화면에 반영되지 않는 상태.

---

## 2. 현재 문제 요약

| 항목 | DB 상태 | 현재 화면 |
|------|---------|----------|
| `genres[]` | ✅ 영문("Strategy" 등) | ❌ GameCard 미표시 |
| `rank` | ✅ 저장 | ❌ 미표시, 정렬도 avg_rating 기준 |
| `rating_count` | ✅ 저장 | ❌ 카드 미표시 |
| 장르 필터 | — | ❌ FilterChips에 없음, FilterSheet 연동 안 됨 |
| 필터 로직 | — | ❌ 검색어만 작동, 나머지 필터 미적용 |
| `min/max_players` | ⚠️ 전부 default(1,4) | ✅ 표시되나 실제값 아님 (Tier 2 전까지 유지) |

---

## 3. 구현 계획

### Step 1 — 상수 정리 (`src/constants/index.ts`)

**영문→한국어 장르 매핑 추가** (DB 값 기준 8개):

| 영문 (DB 저장값) | 한국어 (표시값) |
|----------------|--------------|
| Strategy | 전략 |
| Family | 가족 |
| Party | 파티 |
| Abstract | 추상 |
| Thematic | 테마 |
| War | 워게임 |
| Children | 어린이 |
| Customizable | 커스텀 |

**GENRE_OPTIONS 형식 변경**: 한국어 string[] → `{label, value}[]`  
DB에 없는 항목 제거: "추리", "협력", "경제", "카드", "주사위", "덱빌딩"

---

### Step 2 — GameCard 개선 (`src/components/game/GameCard.tsx`)

추가 표시 항목:
1. **장르 뱃지** — genres[]를 한국어로 변환, 최대 2개 표시
2. **평가수** — 평점 옆에 `(N명)` 형태로 표시
3. **순위 뱃지** — rank 있으면 썸네일 좌상단에 `#N` 표시

---

### Step 3 — FilterChips 장르 칩 추가 (`src/components/game/FilterChips.tsx`)

GENRE_OPTIONS 8개를 필터 칩으로 추가.  
`GameFilter`에 `genre?: string` 단일 선택 필드 추가.

---

### Step 4 — FilterSheet 장르 연동 (`src/components/game/FilterSheet.tsx`)

GENRE_OPTIONS 형식 변경(`{label, value}[]`)에 맞춰 표시/선택 로직 업데이트.

---

### Step 5 — 탐색 페이지 fetch + 필터/정렬 구현 (`src/app/(main)/games/page.tsx`)

- **fetch 정렬 변경**: avg_rating → rank 오름차순 (인기순)
- **필터 로직 추가**: 검색어 + 장르 필터 적용
- **정렬 로직 추가**: 인기순/평점순/가격순/최신순

> `min/max_players`는 전부 default(1,4)이므로 player 필터 로직은 Tier 2 완료 후 추가.

## 5. 검증 방법

1. `http://localhost:3000/games` 접속
2. 게임 카드에 장르 뱃지(한국어), 평가수, 순위(`#N`) 표시 확인
3. FilterChips 장르 칩 클릭 → 해당 장르 게임만 표시 확인
4. 정렬 "인기순" → rank 1번 게임(Brass: Birmingham) 첫 번째 확인
5. 정렬 "평점 높은순" → avg_rating 높은 게임 상단 확인
6. FilterSheet 장르 선택 → 필터링 확인
