# 인터랙션 & 마이크로 UX 정의

> 애니메이션, 피드백, 로딩, 에러 등 전체 화면에 적용되는 인터랙션 규칙

---

## 1. 애니메이션 규칙

### 기본 원칙
- 모든 애니메이션은 **목적이 있어야** 함 (장식용 X)
- `prefers-reduced-motion` 미디어 쿼리 존중 (접근성)
- 300ms를 넘지 않음 (느려지면 답답)

### 트랜지션 속도 체계

| 토큰 | 속도 | 이징 | 용도 |
|------|------|------|------|
| `fast` | 100ms | ease-out | 색상 변경, 토글 |
| `normal` | 150ms | ease-in-out | hover, 포커스, 탭 전환 |
| `moderate` | 200ms | ease-out | 카드 hover, 별점 선택 |
| `slow` | 250ms | ease-out | 페이지 전환, 슬라이드 |
| `sheet` | 300ms | ease-out | 바텀시트, 모달 |

## 2. 페이지 전환

### 모바일

| 전환 | 애니메이션 |
|------|-----------|
| 탭 간 이동 (홈↔탐색↔내기록↔프로필) | 페이드 (150ms) |
| 목록 → 상세 (게임 카드 탭 등) | 우측에서 슬라이드 인 (250ms) |
| 상세 → 뒤로 (← 버튼) | 좌측으로 슬라이드 아웃 (250ms) |
| 모달 열기 | 오버레이 페이드 + 스케일 (0.95→1.0, 200ms) |
| 바텀시트 열기 | 하단에서 슬라이드 업 (300ms) |
| 바텀시트 닫기 | 하단으로 슬라이드 다운 (250ms) 또는 드래그 |

### 데스크톱

| 전환 | 애니메이션 |
|------|-----------|
| 페이지 이동 | 페이드 (150ms) |
| 모달 열기 | 오버레이 페이드 + 스케일 (200ms) |
| 드롭다운 열기 | 페이드 + 슬라이드 다운 (150ms) |

## 3. 컴포넌트 인터랙션

### 3.1 버튼

| 상태 | 시각 변화 |
|------|-----------|
| default | 기본 스타일 |
| hover | 배경 어두워짐 (primary-500 → primary-600), 150ms |
| active/pressed | 배경 더 어두워짐 (primary-700), scale(0.98), 100ms |
| disabled | opacity 0.5, 커서 not-allowed |
| loading | 텍스트 → 스피너(16px) 교체, 클릭 불가 |

```
기본:     [ 구매하기 → ]
로딩:     [    ⟳      ]  ← 스피너 회전
비활성:   [ 구매하기 → ]  ← 흐리게
```

### 3.2 카드 (게임 카드)

| 상태 | 시각 변화 |
|------|-----------|
| default | shadow-sm |
| hover (데스크톱) | translateY(-2px), shadow-md, 200ms |
| active/pressed | scale(0.98), 100ms |

### 3.3 별점

```
선택 전:    ☆  ☆  ☆  ☆  ☆
호버 중:    ★  ★  ★  ☆  ☆   ← 3번째 별까지 미리보기 (accent-200)
선택 후:    ★  ★  ★  ☆  ☆   ← accent-400, 선택된 별 scale(1.15→1.0) 200ms
```

| 동작 | 설명 |
|------|------|
| 호버 (데스크톱) | 마우스 위치까지 별 채워짐 (미리보기, 연한 색) |
| 탭/클릭 | 해당 별까지 확정, 선택된 별 바운스 |
| 재선택 | 같은 별 탭 → 해제 |

### 3.4 좋아요 / 북마크

```
비활성:   ♡ (outline, gray-400)
활성화:   ♥ (filled, error 빨강)     ← scale(1.2→1.0) 200ms, 바운스
해제:     ♡ (outline)                ← scale(0.8→1.0) 150ms
```

북마크도 동일 패턴 (🔖 outline ↔ filled, primary-500)

### 3.5 토글 스위치

```
OFF:   [○        ]  gray-200 배경
ON:    [        ●]  primary-500 배경

전환: 250ms, 스프링 이징
```

### 3.6 필터 칩

```
미선택:  [인원수 ▾]     gray-100 배경
선택됨:  [2~4인 ✕]     primary-100 배경, primary-600 텍스트

전환: 150ms 페이드
```

### 3.7 탭 전환

```
[컬렉션]   [로그]   [평점]   [북마크]
 ━━━━━━

활성 탭 하단 바: 좌우 슬라이드 (200ms, ease-out)
콘텐츠: 페이드 (150ms)
```

## 4. 로딩 상태

### 4.1 스켈레톤 UI (최초 로딩)

```
게임 카드:
┌──────────────────┐
│ ░░░░░░░░░░░░░░░░ │
│ ░░░░░░░░░░░░░░░░ │  ← gray-100, 펄스 애니메이션
│ ░░░░░░░░░░░░░░░░ │     (opacity 0.5 ↔ 1.0, 1.5s)
├──────────────────┤
│ ░░░░░░░░░░       │
│ ░░░░░░           │
└──────────────────┘

리스트 아이템:
┌─────────────────────────────────┐
│ ░░  ░░░░░░░░░░░░    ░░░░       │
├─────────────────────────────────┤
│ ░░  ░░░░░░░░░       ░░░░░      │
└─────────────────────────────────┘
```

| 규칙 | 설명 |
|------|------|
| 형태 | 실제 콘텐츠와 같은 레이아웃 |
| 색상 | gray-100 |
| 애니메이션 | opacity 펄스 (0.5 ↔ 1.0), 1.5초 주기 |
| 개수 | 화면에 보이는 만큼 (게임 카드 4~6개, 리스트 3~5개) |

### 4.2 무한 스크롤

```
목록 끝 도달 시:
┌─────────────────────────────────┐
│             ⟳                   │  ← 스피너 (24px), 중앙
│        불러오는 중...             │  ← caption, gray-400
└─────────────────────────────────┘

더 이상 데이터 없을 때:
┌─────────────────────────────────┐
│     모든 게임을 다 봤어요 🎲      │  ← caption, gray-400
└─────────────────────────────────┘
```

### 4.3 당겨서 새로고침 (모바일)

```
↓ 당기는 중:    스피너 아이콘 회전, 점점 커짐
↓ 임계점 도달:  햅틱 피드백, 아이콘 색상 primary-500으로 변경
↓ 놓음:        상단에서 스피너 회전 → 완료 시 사라짐
```

### 4.4 버튼 로딩

```
저장 전:  [ 저장하기 ]
저장 중:  [    ⟳    ]  ← 스피너로 교체, 버튼 비활성화
완료:     [    ✓    ]  ← 0.5초 체크 표시 후 다음 화면 이동
```

## 5. 토스트 메시지

### 유형

| 유형 | 아이콘 | 색상 | 예시 |
|------|--------|------|------|
| 성공 | ✓ | success (#10B981) | "기록이 저장되었습니다" |
| 에러 | ✕ | error (#EF4444) | "저장에 실패했습니다. 다시 시도해주세요" |
| 정보 | ℹ | primary (#3B82F6) | "외부 사이트로 이동합니다" |
| 언두 | ↩ | gray-900 | "북마크가 해제되었습니다 [되돌리기]" |

### 동작

```
모바일:
┌─────────────────────────────────┐
│  ✓ 기록이 저장되었습니다           │  ← 탭바 위 16px
└─────────────────────────────────┘

데스크톱:
                  ┌──────────────────────┐
                  │  ✓ 기록이 저장되었습니다 │  ← 우상단 24px
                  └──────────────────────┘
```

| 요소 | 규격 |
|------|------|
| 진입 | 하단에서 슬라이드 업 (모바일) / 우측에서 슬라이드 인 (데스크톱), 200ms |
| 노출 | 3초 |
| 퇴장 | 페이드 아웃 (200ms) |
| 스와이프 | 모바일에서 좌/우 스와이프로 즉시 닫기 |
| 중첩 | 최대 3개, 위로 쌓임 |
| 언두 버튼 | 우측, 탭 시 작업 취소 + 토스트 닫힘 |

## 6. 모달 & 바텀시트

### 모달 (데스크톱 + 모바일 일부)

```
┌─ 오버레이 (black 50%) ─────────────┐
│                                     │
│      ┌───────────────────┐         │
│      │  제목          ✕   │         │
│      │                   │         │
│      │  콘텐츠            │         │
│      │                   │         │
│      │  [취소] [확인]     │         │
│      └───────────────────┘         │
│                                     │
└─────────────────────────────────────┘
```

| 요소 | 규격 |
|------|------|
| 오버레이 | black 50%, 탭 시 닫힘 |
| 모달 | max 480px, radius 12px, shadow-lg |
| 진입 | 오버레이 페이드(200ms) + 모달 스케일(0.95→1.0, 200ms) |
| 퇴장 | 역방향 (150ms) |
| ESC | 닫힘 (데스크톱) |

### 바텀시트 (모바일)

```
┌─ 오버레이 ──────────────────────────┐
│                                     │
│                                     │
│  ┌─────────────────────────────┐   │
│  │  ── (드래그 핸들 32×4px)      │   │
│  │                             │   │
│  │  콘텐츠                      │   │
│  │                             │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

| 요소 | 규격 |
|------|------|
| 드래그 핸들 | 32×4px, gray-300, radius 2px, 중앙 |
| 최대 높이 | 화면 85% |
| 진입 | 하단에서 슬라이드 업 (300ms) |
| 닫기 | 아래로 드래그 (임계점 30%) / 오버레이 탭 / ✕ 버튼 |
| 스냅 포인트 | 콘텐츠 높이에 맞춤, 길면 중간 스냅 |

## 7. 폼 인터랙션

### 인풋 필드 상태

| 상태 | 보더 | 배경 | 기타 |
|------|------|------|------|
| default | gray-200 | white | |
| focus | primary-500 (2px) | primary-50 | 라벨 primary-500 |
| error | error (2px) | error-50 | 에러 메시지 표시 |
| disabled | gray-100 | gray-50 | 텍스트 gray-400 |

### 유효성 검사 타이밍

| 필드 | 체크 시점 |
|------|-----------|
| 이메일 | blur 시 형식 체크, 제출 시 중복 체크 |
| 비밀번호 | 입력 중 실시간 (8자 이상 충족 시 ✓) |
| 비밀번호 확인 | 입력 중 실시간 (일치 여부) |
| 닉네임 | 500ms 디바운스 후 중복 체크 |

### 에러 메시지 표시

```
  이메일
  ┌─────────────────────────────┐
  │  abc@                        │  ← error 보더
  └─────────────────────────────┘
  ⚠ 올바른 이메일 형식이 아닙니다     ← error 텍스트 (12px)

  진입: 페이드 + 슬라이드 다운 (150ms)
```

### 키보드 동작 (모바일)

| 상황 | 동작 |
|------|------|
| 인풋 포커스 | 키보드 올라옴, 인풋이 보이도록 스크롤 |
| 키보드 위 버튼 | 제출 버튼이 키보드 바로 위에 고정 (로그인, 회원가입) |
| 바텀시트 내 인풋 | 바텀시트가 키보드 위로 올라감 |

## 9. 에러 처리

### 네트워크 에러

```
┌─────────────────────────────────────┐
│                                     │
│              📡                     │
│    인터넷 연결을 확인해주세요           │
│                                     │
│    [ 다시 시도 ]                     │
│                                     │
└─────────────────────────────────────┘
```

### API 에러

```
일시적 에러 (500):
→ 토스트: "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요"
→ 자동 재시도 1회 (3초 후)

인증 만료 (401):
→ 로그인 화면으로 이동 + 토스트: "다시 로그인해주세요"

권한 없음 (403):
→ 토스트: "접근 권한이 없습니다"

Not Found (404):
→ 404 페이지로 이동
```

### 폼 제출 실패

```
→ 에러 필드 하이라이트 (error 보더)
→ 첫 번째 에러 필드로 스크롤
→ 에러 메시지 표시
→ 제출 버튼 다시 활성화
```

## 10. 외부 링크 (구매하기)

```
구매하기 탭 시:
1. 토스트: "외부 사이트로 이동합니다" (정보, 1.5초)
2. 새 탭으로 외부 쇼핑몰 열림
3. 현재 페이지는 유지
```

## 12. 접근성

| 항목 | 규칙 |
|------|------|
| 색상 대비 | 텍스트 WCAG AA 이상 (4.5:1) |
| 포커스 링 | primary-500, 2px, offset 2px (키보드 네비게이션 시) |
| 대체 텍스트 | 모든 이미지에 alt 속성 |
| 터치 영역 | 최소 44×44px |
| 스크린 리더 | 버튼/링크에 aria-label, 상태 변경 시 aria-live |
| 모션 축소 | prefers-reduced-motion 시 모든 애니메이션 제거 |
