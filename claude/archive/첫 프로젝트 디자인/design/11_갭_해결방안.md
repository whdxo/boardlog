# 피그마 갭 해결 방안

> `10_피그마_갭분석.md` 후속. 각 갭을 어떻게 처리할지 결정 사항 정리.

---

## 결정 사항 요약

| 갭 유형 | 결정 |
|---|---|
| 완전히 빠진 화면 4개 (A-4, U-3, U-4, U-7) | 개발하면서 구현 |
| 반응형 누락 5개 | 개발하면서 구현 |
| GNB 불일치 | 한국어 기준으로 통일 (`홈 탐색 내기록`) |
| 요소 수준 누락 | 아래 항목별 방법 참고 |

---

## 요소 수준 누락 — 항목별 해결 방법

### 1. 게임 상세 — 평점 등록 버튼(E-3) + 평점 모달

피그마 없이 **코드로 바로 구현**.

`06_와이어프레임_탐색.md` 4번에 스펙이 완전히 정의되어 있음:
- 별 크기, 색상, 간격
- 저장/삭제 버튼
- 모바일 → 바텀시트 / 데스크톱 → 중앙 모달

피그마 먼저 만들 필요 없음.

---

### 2. 각 탭별 빈 상태 (empty state)

피그마 없이 **코드로 바로 구현**.

`07_와이어프레임_내기록.md`에 4가지 탭 모두 정의되어 있음. 구조가 동일(아이콘 + 텍스트 + CTA 버튼)해서 컴포넌트 하나로 재사용:

```
<EmptyState icon={...} title="..." description="..." ctaLabel="..." />
```

| 탭 | 아이콘 | 텍스트 | CTA |
|---|---|---|---|
| 컬렉션 | 🎲 | 아직 컬렉션이 비어있어요 | 게임 탐색하기 |
| 로그 | 📝 | 아직 플레이 기록이 없어요 | + 기록하기 |
| 평점 | ⭐ | 아직 평가한 게임이 없어요 | 게임 탐색하기 |
| 북마크 | 🔖 | 북마크한 게임이 없어요 | 게임 탐색하기 |

---

### 3. 비로그인 상태 UI

두 가지로 분리해서 처리.

**GNB 비로그인 상태**
`04_와이어프레임_공통.md` 기준. Next.js 레이아웃에서 세션 유무로 분기:
- 로그인: 알림 🔔 + 프로필 드롭다운
- 비로그인: `[로그인]` (Ghost 버튼) + `[회원가입]` (Primary sm 버튼)

**내기록/프로필 탭 비로그인 유도 화면**
`04_와이어프레임_공통.md` 8번에 인라인 카드 스펙 있음. 인증 가드 컴포넌트에서 렌더링:

```
<LoginPrompt
  icon="🔒"
  title="로그인이 필요한 기능이에요"
  description="로그인하고 기록을 시작하세요"
/>
```

---

### 4. 토스트, 스켈레톤, 로그아웃 모달 등 공통 컴포넌트

UI 라이브러리 컴포넌트로 처리. 피그마 별도 제작 불필요.

| 컴포넌트 | 방법 | 스펙 적용 |
|---|---|---|
| **토스트** | shadcn/ui `sonner` 또는 `toast` 사용 | 노출 3초, 모바일 하단/데스크톱 우상단, 성공(초록)/에러(빨강) |
| **스켈레톤** | shadcn/ui `Skeleton` 사용 | 게임 카드, 리스트 아이템 레이아웃에 맞게 조합 |
| **로그아웃 모달** | shadcn/ui `AlertDialog` 사용 | 텍스트 "로그아웃 하시겠어요?" + 취소/로그아웃 |
| **삭제 확인 모달** | shadcn/ui `AlertDialog` 재사용 | 텍스트 "이 기록을 삭제할까요?" + 취소/삭제(error 컬러) |

스펙 상세는 `04_와이어프레임_공통.md` 참고.

---

## 결론

요소 수준 누락은 **와이어프레임 스펙이 충분하거나 UI 라이브러리로 커버**되므로, 피그마 보완을 기다리지 않고 개발하면서 처리한다.
