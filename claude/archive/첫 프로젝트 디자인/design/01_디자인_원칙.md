# 디자인 원칙 & 방향성

---

## 한 줄 요약

> 보드게임을 발견하고, 후기를 확인하고, 바로 구매할 수 있는 깔끔한 플랫폼

---

## 1. 디자인 키워드

| 키워드 | 설명 |
|--------|------|
| **Clean** | 여백 충분, 정보 밀도 낮게, 한눈에 읽히는 구조 |
| **Playful** | 둥근 모서리, 부드러운 애니메이션, 아이콘 활용으로 재미 요소 |
| **Trustful** | 푸른 톤 기반 신뢰감, 일관된 컴포넌트, 예측 가능한 인터랙션 |

### 하지 않을 것
- 보드라이프처럼 빽빽한 게시판 레이아웃
- 과도한 장식, 그라데이션, 그림자
- 정보 과부하 (한 화면에 너무 많은 요소)

## 3. 타이포그래피

### 폰트 선택

| 용도 | 폰트 | 이유 |
|------|------|------|
| 본문 | **Pretendard** | 한국어 가독성 최고, 무료, 웹폰트 지원 |
| 대안 | Noto Sans KR | Google Fonts 지원, 접근성 좋음 |
| 숫자/영문 강조 | **Inter** | Pretendard와 조화, 숫자 가독성 |

### 크기 체계

```
Display    32px / Bold    — 페이지 타이틀 (거의 안 씀)
Heading 1  24px / Bold    — 섹션 제목
Heading 2  20px / SemiBold — 카드 제목, 게임명
Heading 3  16px / SemiBold — 소제목
Body       14px / Regular  — 본문
Caption    12px / Regular  — 보조 텍스트, 날짜, 태그
```

## 4. 레이아웃 원칙

### 반응형 (웹 + 모바일)

```
Mobile       ~767px    1컬럼, 하단 탭바
Tablet       768~1023px  2컬럼
Desktop      1024px~     최대 1080px 컨테이너, 상단 GNB, 양쪽 여백 넉넉
```

### 네비게이션

| 디바이스 | 방식 |
|---------|------|
| Mobile | 하단 탭바 (홈 · 탐색 · 커뮤니티 · 내기록 · 프로필) |
| Desktop | 상단 GNB + 우측 프로필 드롭다운 |

### 간격 체계

```
4px 단위 (4, 8, 12, 16, 24, 32, 48)
카드 내부 패딩: 16px
카드 간 간격: 16px (모바일) / 24px (데스크톱)
섹션 간 간격: 32px (모바일) / 48px (데스크톱)
```

## 5. 컴포넌트 스타일 방향

| 요소 | 스타일 |
|------|--------|
| 카드 | `border-radius: 12px`, 얇은 보더 or 미세한 그림자 |
| 버튼 | `border-radius: 8px`, Primary 배경 + 흰 텍스트 |
| 인풋 | `border-radius: 8px`, 회색 보더, 포커스 시 파란 보더 |
| 태그/뱃지 | `border-radius: 9999px` (pill), 연한 배경 + 진한 텍스트 |
| 아이콘 | Lucide Icons (shadcn/ui 기본, 선형 스타일) |
| 이미지 | `border-radius: 8px`, 게임 썸네일은 3:4 비율 |

### 플레이풀 요소 (과하지 않게)
- 빈 상태(empty state)에 일러스트 or 이모지
- 별점 인터랙션에 작은 바운스 애니메이션
- 플레이 로그 작성 완료 시 축하 효과 (confetti는 X, 체크 아이콘 정도)
- 게임 카드 hover 시 살짝 위로 떠오르는 효과

## 6. 레퍼런스 분석

### 보드라이프 — 기능은 참고, UX는 반면교사

| 요소 | 보드라이프 문제점 | BoardLog 방향 |
|------|-----------------|---------------|
| 네비게이션 | 메뉴 20개 이상, 분산 | 탭 4개로 집중 |
| 게임 목록 | 텍스트 테이블 나열 | 카드형 그리드, 썸네일 중심 |
| 게임 상세 | 정보가 여러 탭에 분산 | 한 페이지에 정보 + 평점 + 구매 링크 |
| 구매 | 자체 쇼핑몰 (복잡) | 외부 링크로 간단 연결 (MVP) |
| 모바일 | 반응형이지만 데스크톱 우선 | 모바일 우선 설계 |

### Letterboxd — 기록 UX 참고

- 별점 반개 단위 (★★★½) → 직관적
- 영화 포스터 중심 그리드 → 게임 썸네일 그리드로 적용
- 기록 시 간단한 폼 (날짜 + 별점 + 한줄메모) → 플레이 로그에 적용
- 프로필 = 내가 본 영화 모아보기 → 내가 한 게임 모아보기

### Goodreads — 컬렉션 참고

- "읽고 싶은 / 읽는 중 / 읽음" 상태 분류 → "위시 / 보유 / 플레이완료"
- 별점 + 짧은 리뷰 구조 → Phase 2 텍스트 리뷰에 적용
