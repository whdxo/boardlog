# 차별화 전략

## 보드라이프 vs BoardLog

| 구분 | 보드라이프 | BoardLog |
|-----|-----------|---------|
| 구조 | 게시판형 (올드) | 카드형 (모바일 중심) |
| 핵심 | 커뮤니티 + DB | **구매 + 탐색 + 기록** |
| 구매 | 자체 쇼핑 (복잡) | MVP: 외부 링크, Phase 2: 자체 결제 |
| 기록 | 간단한 컬렉션 | 플레이 로그 (누구와, 언제, 어디서) |
| 추천 | 없음 | 취향 기반 추천 (Phase 2) |
| UX | 복잡, 분산 | 단순, 집중 |
| 데이터 | 유저 참여형 (정확도 낮음) | 큐레이션 + BGG 연동 |

## 핵심 차별화 포인트 3가지

### 1. 구매까지 연결되는 탐색
보드게임을 발견 → 후기/평점 확인 → 바로 구매까지 한 흐름으로.
- 게임 상세 페이지에서 바로 최저가 구매 링크 제공
- MVP는 네이버 쇼핑 등 외부 연결, Phase 2에서 자체 결제
- 후기와 평점이 구매 결정을 돕는 구조

### 2. 플레이 기록 (보드게임 다이어리)
단순 컬렉션 관리가 아니라 "보드게임 다이어리"
- **언제, 누구와, 어디서** 플레이했는지 기록
- 플레이 메모 및 감상 기록
- 이게 쌓이면 유저가 못 떠남 (락인)

### 3. 깔끔한 UX (모바일 우선)
보드라이프의 가장 큰 약점은 UX.
- 카드형 게임 탐색
- 복잡한 메뉴 없이 탭 4개로 모든 것
- 구매까지 최소 클릭으로 도달

## 타겟 유저

### 메인 타겟
- 20~40대 보드게임 취미 유저
- 보드게임을 사고 싶은 사람
- 다른 사람의 후기/평점을 보고 구매를 결정하는 사람
- 플레이 기록을 남기고 싶은 사람

### 세컨더리 타겟
- 보드게임 입문자 (뭘 사야 할지 모르는 사람)
- 보드게임 카페 운영자 (Phase 2+)
- 보드게임 제작자 (펀딩 기능으로 유입, Phase 3)

## 경쟁 서비스 분석

| 서비스 | 강점 | 약점 | 우리 포지션 |
|-------|------|------|------------|
| 보드라이프 | 트래픽, 데이터, 쇼핑 | UX, 기능 분산 | UX + 기록 + 구매 통합 |
| BoardGameGeek | 글로벌 DB, 데이터 | 한국어 없음, UX 최악 | 국내 특화 |
| 네이버 쇼핑 | 최저가, 트래픽 | 보드게임 특화 없음 | 보드게임 전문 큐레이션 |
| 쿠팡 | 빠른 배송 | 후기 신뢰도 낮음 | 전문 유저 후기 기반 |
| 당근마켓 | 중고거래 | 보드게임 특화 없음 | Phase 2에서 중고 공략 |
