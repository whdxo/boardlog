| 서비스 | 역할 | 선택 이유 |
|--------|------|----------|
| **Vercel** | 프론트엔드 + API Routes 호스팅 | Next.js 최적화, 무료 티어 넉넉 |

## 4. Supabase 설정

### 프로젝트 생성

| 항목 | 값 |
|------|-----|
| Organization | 개인 |
| Project Name | boardlog |
| Database Password | 강력한 비밀번호 생성 |
| Region | Northeast Asia (Seoul) |
| Pricing Plan | Free |

### DB (PostgreSQL)

- Prisma로 스키마 관리 → `npx prisma migrate deploy`로 마이그레이션
- Connection Pooling 사용 (Supabase에서 기본 제공)
- `DATABASE_URL`은 Supabase 대시보드 → Settings → Database에서 복사

### 인증 (Supabase Auth)

| 제공자 | 설정 |
|--------|------|
| 이메일 | 기본 활성화, 이메일 확인 on |
| 카카오 | Kakao Developers에서 앱 생성 → Client ID/Secret 입력 |
| Apple | Apple Developer에서 Service ID 생성 → Key 입력 |
| Google | Google Cloud Console에서 OAuth 2.0 → Client ID/Secret 입력 |

### 스토리지 (이미지)

| 버킷 | 용도 | 정책 |
|------|------|------|
| `profiles` | 프로필 이미지 | 인증 유저만 업로드, 공개 읽기 |
| `banners` | 홈 배너 이미지 | 관리자만 업로드, 공개 읽기 |
| `games` | 게임 썸네일 | 관리자만 업로드, 공개 읽기 |

이미지 정책:
- 프로필: 최대 2MB, JPG/PNG/WebP
- 게임/배너: 최대 5MB, JPG/PNG/WebP

```

Stage 2 — 초기 성장 (~수천 유저)
  Vercel Pro + Supabase Pro = $45/월

Stage 3 — 본격 성장 (~만 유저 이상)
  Vercel Pro + Supabase Pro + CDN(Cloudflare) = $50~100/월
```

## 6. 배포 플로우

### Git → Vercel 자동 배포

```
코드 푸시 (GitHub)
       │
       ▼
  ┌──────────┐
  │  Vercel   │
  │  자동 빌드  │  ← main 브랜치 푸시 시 자동 트리거
  └──────────┘
       │
       ├── 빌드 성공 → 프로덕션 배포
       │
       └── PR 생성 시 → Preview 배포 (PR별 임시 URL)
```

### GitHub Actions (CI)

PR 생성/업데이트 시 자동으로 코드 품질 체크. 배포는 Vercel이 담당.

```
PR 생성 / 업데이트
       │
       ▼
  ┌──────────────────────────────────┐
  │  GitHub Actions                   │
  │                                   │
  │  1. 린트 (ESLint)                  │
  │  2. 타입 체크 (tsc --noEmit)       │
  │  3. 빌드 테스트 (npm run build)    │
  │  4. Prisma 스키마 검증             │
  └──────────────────────────────────┘
       │
       ├── 전부 통과 ✅ → 머지 가능
       │
       └── 실패 ❌ → 머지 차단
```

워크플로우 파일: `.github/workflows/ci.yml`

```yaml
name: CI

on:
  pull_request:
    branches: [main, develop]

jobs:
  check:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint

      - name: Type check
        run: npx tsc --noEmit

      - name: Build
        run: npm run build
        env:
          NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}
          NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}

      - name: Prisma validate
        run: npx prisma validate
```

| 체크 | 역할 | 실패 시 |
|------|------|---------|
| **Lint** | 코드 스타일, 에러 패턴 검출 | 코드 수정 필요 |
| **Type check** | TypeScript 타입 에러 검출 | 타입 수정 필요 |
| **Build** | 빌드 깨지는지 확인 | import 에러, 환경변수 누락 등 |
| **Prisma validate** | DB 스키마 문법 오류 검출 | 스키마 수정 필요 |

> GitHub 레포 Settings → Branches → Branch protection rules에서 main 브랜치에 "Require status checks to pass" 설정하면 CI 통과 전 머지 차단 가능.

### 브랜치 전략

| 브랜치 | 용도 | 배포 |
|--------|------|------|
| `main` | 프로덕션 | Vercel 프로덕션 자동 배포 |
| `develop` | 개발 통합 | Vercel Preview |
| `feature/*` | 기능 개발 | PR 시 Preview |

### 배포 명령어

```bash
# 로컬 개발
npm run dev

# 빌드 테스트
npm run build

# Prisma 마이그레이션 (DB 스키마 변경 시)
npx prisma migrate dev          # 로컬
npx prisma migrate deploy       # 프로덕션

# Vercel 배포 (자동 — GitHub 푸시만 하면 됨)
git push origin main
```

## 9. 배포 체크리스트

### 최초 배포 시

```
1. GitHub 레포지토리 생성
2. Vercel에 GitHub 레포 연결
3. Supabase 프로젝트 생성 (Seoul 리전)
4. Supabase에서 DB URL 복사
5. Vercel 환경변수 설정 (위 목록 전부)
6. Prisma 마이그레이션 실행
7. 소셜 로그인 키 발급 (카카오, Apple, Google)
8. Supabase Auth에 소셜 제공자 설정
9. Supabase Storage 버킷 생성
10. main 브랜치 푸시 → 자동 배포
11. 배포 URL 확인 (boardlog.vercel.app)
12. Sentry 프로젝트 생성 + DSN 설정
```

### 매 배포 시

```
1. 로컬에서 npm run build 성공 확인
2. DB 마이그레이션 필요 여부 확인
3. 환경변수 변경 여부 확인
4. main에 머지/푸시
5. Vercel 빌드 로그 확인
6. 배포 후 주요 기능 스모크 테스트
```
