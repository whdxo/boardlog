  PlayerStatsTable.tsx       ← 플레이어별 승리/승률/점수 테이블

## 4. DB 쿼리 설계

### 4-1. 컬렉션 카운트

```sql
SELECT status, COUNT(*) as count
FROM collections
WHERE user_id = $1
GROUP BY status
```

### 4-2. 컬렉션 게임 목록

```sql
SELECT g.*, c.status, c.created_at as collected_at
FROM collections c
JOIN games g ON c.game_id = g.id
WHERE c.user_id = $1
  AND ($2 IS NULL OR c.status = $2)  -- 상태 필터
ORDER BY c.created_at DESC           -- 최신순
LIMIT 20 OFFSET $3
```

### 4-3. 게임별 플레이 요약

```sql
-- 첫 플레이 + 총 횟수
SELECT 
  MIN(played_at) as first_played_at,
  COUNT(*) as total_plays
FROM play_logs
WHERE user_id = $1 AND game_id = $2

-- 내 평점
SELECT score FROM ratings
WHERE user_id = $1 AND game_id = $2
```

### 4-4. 점수 통계

```sql
SELECT 
  MAX(pls.score) as highest,
  MIN(pls.score) as lowest,
  ROUND(AVG(pls.score), 1) as average
FROM play_log_scores pls
JOIN play_logs pl ON pls.play_log_id = pl.id
WHERE pl.user_id = $1 AND pl.game_id = $2
```

### 4-5. 최다 승리자

```sql
SELECT pls.player_name, COUNT(*) as wins
FROM play_log_scores pls
JOIN play_logs pl ON pls.play_log_id = pl.id
WHERE pl.user_id = $1 AND pl.game_id = $2 AND pls.is_winner = true
GROUP BY pls.player_name
ORDER BY wins DESC
LIMIT 1
```

### 4-6. 플레이어 통계

```sql
SELECT 
  pls.player_name,
  COUNT(*) as total_plays,
  SUM(CASE WHEN pls.is_winner THEN 1 ELSE 0 END) as wins,
  ROUND(AVG(pls.score), 1) as avg_score,
  MAX(pls.score) as high_score
FROM play_log_scores pls
JOIN play_logs pl ON pls.play_log_id = pl.id
WHERE pl.user_id = $1 AND pl.game_id = $2
GROUP BY pls.player_name
ORDER BY wins DESC
```

### 4-7. 인원별 평점

```sql
SELECT 
  player_count,
  COUNT(*) as play_count,
  ROUND(AVG(rating), 2) as avg_rating
FROM play_logs
WHERE user_id = $1 AND game_id = $2 AND player_count IS NOT NULL
GROUP BY player_count
ORDER BY player_count
```

### 4-8. 장소 통계

```sql
SELECT location, COUNT(*) as play_count
FROM play_logs
WHERE user_id = $1 AND game_id = $2 AND location IS NOT NULL
GROUP BY location
ORDER BY play_count DESC
```

## 5. 구현 순서

### Phase 1: 컬렉션 메인 (내 게임 목록)

1. `CollectionCategoryBar` 컴포넌트 — 상태별 카운트 칩
2. `CollectionList` 컴포넌트 — 게임 카드 그리드 + 상태 필터 + 정렬
3. `useMyCollection` 훅 — Supabase 쿼리 + React Query
4. `/my/page.tsx` 리팩토링 — 위 컴포넌트 적용
5. 게임 추가 플로우 — `+` 버튼 → 검색 → 컬렉션에 추가

### Phase 2: 게임별 기록 상세

1. `GameRecordHeader` — 게임 정보 + 플레이 요약
2. `ScoreStatsTable` — 점수 통계
3. `PlayerStatsTable` — 플레이어 통계
4. `PlayerCountRatings` — 인원별 평점
5. `LocationStats` — 장소 통계
6. `GamePlayLogList` — 플레이 로그 목록
7. `useGameStats` 훅 — 통계 데이터 패칭
8. `/my/games/[id]/page.tsx` — 위 컴포넌트 조합

### Phase 3: 공개 대시보드

1. `/u/[nickname]/page.tsx` — 프로필 + 컬렉션 (읽기 전용)
2. `/u/[nickname]/games/[id]/page.tsx` — 게임별 기록 (읽기 전용)
3. 공유 버튼 — `/my`에서 링크 복사 기능

### Phase 4: 마무리

1. 로딩 스켈레톤 & 빈 상태
2. 무한 스크롤 or 페이지네이션
3. 영상 섹션 placeholder (video_url 마이그레이션은 나중에)

## 6. 선행 조건

| 항목 | 상태 | 영향 |
|------|------|------|
| games 테이블 데이터 | 비어있음 | 빈 상태 UI로 대응, BGG 검색 연동 필요 |
| BGG API 연동 | 미구현 | 게임 추가 시 BGG 검색 → DB 저장 흐름 필요 |
| 인증 (Supabase Auth) | 구현됨 | 사용 가능 |
| collections 테이블 | 존재 | 바로 사용 가능 |
| play_logs / play_log_scores | 존재 | 바로 사용 가능 |
| ratings 테이블 | 존재 | 바로 사용 가능 |

> **핵심**: BGG 검색 → DB 저장 흐름이 있어야 컬렉션에 게임을 추가할 수 있음.
> Phase 1에서 게임 추가 플로우를 구현할 때 BGG API 연동을 최소한으로 함께 구현해야 함.
