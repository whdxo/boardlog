## 6. 미구현 — 상세 후속 작업

### 6-1. games 테이블 데이터 없음 (최우선)

현재 `games` 테이블이 비어있어서 컬렉션에 게임을 추가할 수 없다.
컬렉션/기록/통계 모든 기능이 games 데이터에 의존한다.

| 작업 | 설명 | 관련 파일 |
|------|------|----------|
| BGG 검색 API 연동 | `/xmlapi2/search?query=` → 게임 ID 목록 반환 | 신규: `src/lib/api/bgg.ts` |
| BGG 상세 API 연동 | `/xmlapi2/thing?id=&stats=1` → 게임 상세 반환 | 신규: `src/lib/api/bgg.ts` |
| XML → DB 파싱/저장 | BGG XML 응답을 `games` 테이블에 INSERT | 신규: `src/lib/api/bgg.ts` |
| 게임 검색 UI | 검색어 입력 → 우리 DB 먼저 → 없으면 BGG 검색 → 결과에서 선택 → DB 저장 | 기존: `GameSearchSheet.tsx` 수정 |
| 컬렉션 추가 플로우 | `/my` 컬렉션 탭 `+` 버튼 → 게임 검색 → 상태 선택 → `collections` INSERT | 기존: `CollectionSheet.tsx` 연동 |

> 참고 문서: `claude/docs/api/02_데이터_매핑_활용.md`, `claude/docs/api/04_BGG_API_실제_응답_데이터.md`

### 6-2. /my 페이지 — 기록/평점/셀렉션 탭 Mock → Supabase

현재 `MOCK_LOGS`, `MOCK_RATINGS`, `MOCK_SELECTIONS` 하드코딩 중.

| 탭 | 필요 API 함수 | 쿼리 대상 테이블 |
|-----|-------------|----------------|
| 기록 | `getMyPlayLogs(userId, page)` | `play_logs` JOIN `games` |
| 평점 | `getMyRatings(userId, page)` | `ratings` JOIN `games` |
| 셀렉션 | `getMySelections(userId)` | `selections` + `selection_games` JOIN `games` |

각 탭별 React Query 훅도 추가 필요:
- `useMyPlayLogs(userId)` → 월별 그룹핑
- `useMyRatings(userId)` → 점수순/최신순 정렬
- `useMySelections(userId)` → 목록 + 게임 수

### 6-3. /my 프로필 헤더 — 통계 카운트 Mock → 실 데이터

현재 통계 카드(게임/기록/평점/셀렉션) 값이 Mock이다.

| 통계 | 쿼리 |
|------|------|
| 게임 수 | `collections` COUNT WHERE user_id (이미 `countsData.total`로 가능) |
| 기록 수 | `play_logs` COUNT WHERE user_id |
| 평점 수 | `ratings` COUNT WHERE user_id |
| 셀렉션 수 | `selections` COUNT WHERE user_id |

→ `getMyProfileStats(userId)` API 함수 하나로 통합 가능

### 6-4. 공개 대시보드 — 빠진 기능들

`/u/[nickname]` 페이지가 만들어졌지만 아래가 빠져있다:

| 빠진 기능 | 설명 | 현재 상태 |
|-----------|------|----------|
| 기록/평점/셀렉션 탭 | 현재 컬렉션만 표시. 다른 사람의 기록/평점/셀렉션도 볼 수 있어야 함 | 탭 자체가 없음 |
| 팔로워/팔로잉 수 표시 | `profiles.follower_count`, `following_count` | 표시 안 됨 |
| 팔로우 버튼 | 다른 사람 프로필에서 팔로우/언팔로우 | 미구현 (`follows` 테이블 존재) |
| 통계 카드 | 게임/기록/평점/셀렉션 카운트 | 컬렉션 수만 표시 |
| `/my`에서 공유 링크 | 내 프로필에서 공개 URL 복사 버튼 | `/my/page.tsx`에 없음 |
| CollectionList 클릭 링크 | `isOwner=false`일 때 `/u/[nickname]/games/[id]`로 가야 하는데 현재 `/games/[id]`로 감 | `CollectionList.tsx:61` 수정 필요 |

### 6-5. 컬렉션 CRUD 미연동

컬렉션 데이터를 읽기만 하고, 쓰기(추가/수정/삭제)가 연결되지 않았다.

| 작업 | 설명 | 관련 컴포넌트 |
|------|------|-------------|
| 게임 → 컬렉션 추가 | `collections` INSERT (user_id, game_id, status) | `CollectionSheet.tsx` |
| 상태 변경 | `collections` UPDATE status | `CollectionSheet.tsx` |
| 컬렉션에서 제거 | `collections` DELETE | `CollectionSheet.tsx` |
| 평점 등록/수정 | `ratings` UPSERT (score, comment) | `RatingModal.tsx` |
| collectionStore 동기화 | Supabase 변경 후 Zustand store 업데이트 | `collectionStore.ts` |
| React Query 캐시 무효화 | 변경 후 `invalidateQueries` 호출 | 각 mutation 훅 |

### 6-6. 플레이 기록 CRUD 미연동

`/my/logs/write` 페이지가 존재하지만 실제 Supabase INSERT가 연결되어 있는지 확인 필요.

| 작업 | 설명 |
|------|------|
| 플레이 기록 작성 | `play_logs` INSERT + `play_log_scores` INSERT |
| 기록 수정 | `play_logs` UPDATE + `play_log_scores` DELETE/재INSERT |
| 기록 삭제 | `play_logs` DELETE (CASCADE로 scores도 삭제) |
| 기록 후 통계 갱신 | `getGameStats` React Query 캐시 무효화 |

### 6-7. RLS(Row Level Security) 공개 범위 확인

현재 공개 대시보드는 다른 사람의 데이터를 SELECT하는데, RLS 정책이 이를 허용하는지 실 테스트 필요.

| 테이블 | 공개 읽기 필요 | RLS 확인 필요 |
|--------|--------------|-------------|
| `profiles` | 닉네임/프로필 조회 | 필요 |
| `collections` + `games` | 컬렉션 목록 조회 | 필요 |
| `play_logs` + `play_log_scores` | 게임별 통계 조회 | 필요 |
| `ratings` | 평점 조회 | 필요 |
| `selections` | `is_public=true`인 것만 | 조건부 필요 |

> 스키마에 RLS가 설정되어 있으나, 실제 `anon` 키로 타 유저 데이터 SELECT가 되는지 테스트 안 됨.
