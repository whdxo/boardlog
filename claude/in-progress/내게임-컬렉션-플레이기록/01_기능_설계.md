# 내 게임 — 기능 설계

## 1. 개요

보드게임 컬렉션 관리 + 게임별 플레이 기록/통계 + 공개 프로필 공유 기능.
보드라이프의 "내 게임" 탭을 레퍼런스로 하되, BoardLog 자체 구조에 맞게 설계.

---

## 2. 컬렉션 상태 정의

기존 DB의 8개 상태를 그대로 사용한다. 보드라이프와 비교:

| 보드라이프 | BoardLog (기존) | 비고 |
|-----------|----------------|------|
| 보유 | `owned` | 동일 |
| 팬 | `fan` | 동일 |
| 위시리스트 | `wishlist` | 동일 |
| 즐겨본 게임 | `completed` | "플레이 완료"로 표현 |
| 선주문 | `preorder` | 동일 |
| 방출예정 / 방출 | `selling` | 하나로 통합 |
| (없음) | `lent` | 빌려줌 |
| (없음) | `borrowed` | 빌림 |
| 구독 | — | 미지원 (필요 시 추가) |
| 펀딩 | — | 미지원 (필요 시 추가) |

> 1인 개발이므로 8개로 충분. 나중에 필요하면 CHECK 제약만 수정하면 됨.

## 3. 화면별 BGG 데이터 → UI 매핑

### 3-1. 내 게임 메인 (컬렉션 목록)

```
┌─────────────────────────────────────┐
│ 컬렉션 >    전체 v    🔍    +      │
├─────────────────────────────────────┤
│ 📦 보유  3  │  ⭐ 팬  1  │  ...    │  ← 상태별 카운트
├─────────────────────────────────────┤
│ 내 전체 컬렉션          최신순 v    │
├─────────────────────────────────────┤
│ [썸네일]  카탄                      │
│          Catan              ⋮      │
│ [썸네일]  아줄                      │
│          Azul               ⋮      │
└─────────────────────────────────────┘
```

| UI 요소 | DB 소스 | BGG 원본 필드 |
|---------|---------|--------------|
| 썸네일 | `games.thumbnail` | `thumbnail` URL |
| 한국어 이름 | `games.title` | `name[alternate]` 한국어 |
| 영문 이름 | `games.title_en` | `name[primary]` |
| 상태별 카운트 | `collections` COUNT GROUP BY status | — (유저 데이터) |
| 컬렉션 상태 | `collections.status` | — (유저 데이터) |

**데이터 흐름:**
```
collections (user_id = 나) 
  → JOIN games ON collections.game_id = games.id
  → 상태별 필터링 & 정렬
```

### 3-2. 게임별 기록 상세

```
┌─────────────────────────────────────┐
│  플레이 >     게임 v               │
├─────────────────────────────────────┤
│ [이미지]  게임 오브 게임즈          │
│          Game of Games             │
│          3-6명 · 40-60분           │
├─────────────────────────────────────┤
│ 첫 플레이:     2026-04-08          │
│ 총 플레이 횟수: 5회                │
│ 내 평가 평점:  ★4.50              │
├─────────────────────────────────────┤
│ 최고점수 | 최저점수 | 평균점수 | 최다승리 │
│   95    |   42    |  68.5  | 홍길동   │
│         |         |        |  3회    │
├─────────────────────────────────────┤
│ ▶ 플레이어 통계                    │
│ 이름   | 승리 | 승률  | 평점 | 최고  │
│ 홍길동 |  3  | 60%  | 4.2 |  95  │
│ 김철수 |  2  | 40%  | 3.8 |  88  │
├─────────────────────────────────────┤
│ ▶ 인원별 평점                      │
│ 인원   | 플레이수 | 내 평점         │
│ 3인플  |    2    |  4.0           │
│ 4인플  |    3    |  4.5           │
├─────────────────────────────────────┤
│ ▶ 장소 통계                        │
│ 보드게임카페A — 3회                │
│ 집 — 2회                          │
├─────────────────────────────────────┤
│ ▶ 플레이 로그                      │
│ 수요일 2026-04-08                  │
│ ★5.00/5 · 4인플              >    │
└─────────────────────────────────────┘
```

| UI 섹션 | 데이터 소스 | 계산 방법 |
|---------|-----------|----------|
| 게임 정보 헤더 | `games` 테이블 | 직접 필드 (thumbnail, title, players, playtime) |
| 첫 플레이 | `play_logs` | `MIN(played_at)` WHERE game_id & user_id |
| 총 플레이 횟수 | `play_logs` | `COUNT(*)` WHERE game_id & user_id |
| 내 평가 평점 | `ratings` | `score` WHERE game_id & user_id (10점→5점 변환) |
| 최고/최저/평균 점수 | `play_log_scores` JOIN `play_logs` | `MAX/MIN/AVG(score)` |
| 최다 승리 | `play_log_scores` JOIN `play_logs` | `is_winner=true` GROUP BY player_name |
| 플레이어 통계 | `play_log_scores` JOIN `play_logs` | GROUP BY player_name → 승리/승률/점수 집계 |
| 인원별 평점 | `play_logs` | GROUP BY player_count → COUNT, AVG(rating) |
| 장소 통계 | `play_logs` | GROUP BY location → COUNT |
| 플레이 로그 | `play_logs` | ORDER BY played_at DESC, 페이지네이션 |

### 3-3. 공개 대시보드

다른 유저의 컬렉션과 통계를 볼 수 있는 공개 페이지.

```
URL: /u/{닉네임}

표시 내용:
- 유저 프로필 (닉네임, 프로필 이미지, 자기소개)
- 컬렉션 목록 (읽기 전용, 수정 버튼 없음)
- 게임별 기록 상세 링크 → /u/{닉네임}/games/{id}
```

> 기존 RLS가 collections, play_logs, ratings에 public SELECT을 허용하므로 추가 설정 불필요.

### 3-4. 게임 설명 영상 (참고)

자체 제작 영상을 게임 상세에 넣을 예정.
- `games` 테이블에 `video_url TEXT` 컬럼 추가 (나중에)
- `/games/[id]` 상세 페이지에 YouTube embed 섹션
- 당장은 구현하지 않고, 구조만 인지

## 4. 게임 데이터 표현 방식 (BGG 기준)

BGG에서 가져올 수 있는 데이터로 게임을 어떻게 표현할지:

### 게임 카드 (목록)
```
[썸네일]  카탄 (Catan)
          ★3.6 · 3~4인 · 60~120분
```
- 평점: BGG average ÷ 2 → 5점 만점
- 인원: `min_players`~`max_players` + "인"
- 시간: `play_time_min`~`play_time_max` + "분"

### 게임 상세 헤더
```
[고해상도 이미지]
카탄
Catan
3~4인 · 60~120분 · 10세+
디자이너: Klaus Teuber
퍼블리셔: Kosmos
출시: 1995
```
- 이미지: BGG `image` (고해상도)
- 이름: `title` (한국어) + `title_en` (영문)
- 정보 칩: 인원, 시간, 연령
- 메타: designer, publisher, year_published

### 태그
```
[전략] [협상] [경제]          ← boardgamecategory → genre 태그
[주사위] [교역] [모듈러보드]   ← boardgamemechanic → mechanic 태그
```

### 평점/통계
```
★3.6 (BGG 7.15)  ·  98,234명 평가  ·  BGG #423
```
- 자체 평점 (유저 10개+): 우선 표시
- BGG 평점: 보조 표시 (10점 기준 함께 노출)

## 5. 빈 데이터 처리

games 테이블이 아직 비어있으므로:

| 상황 | 대응 |
|------|------|
| 컬렉션 비어있음 | EmptyState: "게임을 추가해보세요" + 검색 버튼 |
| 플레이 기록 없음 | EmptyState: "첫 플레이를 기록해보세요" + 기록 버튼 |
| 점수 데이터 없음 | 점수 통계 섹션 "-" 표시 |
| 장소 데이터 없음 | "장소별 통계가 없어요." 텍스트 |
| 게임 검색 결과 없음 | BGG API 검색 → 결과를 DB에 저장 후 표시 |
