# 권한 검토 — 로그인 vs 비회원

## 1. 현재 상태 요약

Supabase RLS(Row Level Security)로 모든 테이블에 권한이 설정되어 있다.
비회원은 **읽기만 가능**, 쓰기는 전부 로그인 필요.

---

## 2. 비회원이 볼 수 있는 것

| 페이지 | 볼 수 있는 데이터 | 비고 |
|--------|------------------|------|
| `/games` | 게임 목록 전체 | 검색, 필터 가능 |
| `/games/{id}` | 게임 상세 (이미지, 설명, 평점 등) | |
| `/u/{닉네임}` | 다른 사람의 컬렉션, 통계 | 공개 프로필 |
| `/u/{닉네임}/games/{id}` | 다른 사람의 게임별 기록 | |
| `/community` | 게시글 목록, 상세 | 읽기만 |
| `/local` | 보드게임 카페/매장 | |
| `/shop` | 상품 목록 | |
| `/used` | 중고거래 글 | |
| 홈 (`/`) | 배너, 인기 게임, 추천 게임 | |

> DB의 거의 모든 테이블에 `FOR SELECT USING (true)` 정책이 있어서
> **비회원도 모든 데이터를 읽을 수 있다.**

---

## 3. 로그인해야 할 수 있는 것

| 기능 | 필요 테이블 | RLS 정책 |
|------|-----------|---------|
| 컬렉션에 게임 추가/수정/삭제 | `collections` | `auth.uid() = user_id` |
| 게임 평점 매기기 | `ratings` | `auth.uid() = user_id` |
| 플레이 기록 작성 | `play_logs`, `play_log_scores` | `auth.uid() = user_id` |
| 셀렉션 만들기 | `selections`, `selection_games` | `auth.uid() = user_id` |
| 게시글 작성/수정/삭제 | `posts` | `auth.uid() = user_id` |
| 댓글 작성 | `comments` | `auth.uid() = user_id` |
| 게시글 좋아요 | `post_likes` | `auth.uid() = user_id` |
| 게임 북마크 | `bookmarks` | `auth.uid() = user_id` |
| 팔로우/언팔로우 | `follows` | `auth.uid() = follower_id` |
| 프로필 수정 | `profiles` | `auth.uid() = id` |
| 이미지 업로드 | Storage | `auth.role() = 'authenticated'` |
| 알림 확인 | `notifications` | `auth.uid() = user_id` |

## 4. 내 게임 기능에서의 권한

### 4-1. `/my` 페이지

| 상태 | 동작 |
|------|------|
| 비회원 | "로그인하고 내 게임을 관리해요" LoginPrompt 표시 ✅ |
| 회원 | 컬렉션/기록/평점/셀렉션 탭 표시 ✅ |

### 4-2. `/my/games/{id}` 게임별 기록

| 상태 | 동작 |
|------|------|
| 비회원 | "로그인이 필요해요" LoginPrompt 표시 ✅ |
| 회원 | 자기 통계만 표시 ✅ |

### 4-3. `/u/{닉네임}` 공개 프로필

| 상태 | 동작 |
|------|------|
| 비회원 | 다른 사람 컬렉션 볼 수 있음 ✅ |
| 회원 | 동일 ✅ |

> RLS가 collections, play_logs, ratings에 공개 SELECT을 허용하므로
> 비회원도 다른 사람의 데이터를 정상적으로 볼 수 있다.

### 4-4. `/u/{닉네임}/games/{id}` 공개 게임 기록

| 상태 | 동작 |
|------|------|
| 비회원 | 다른 사람 통계/로그 볼 수 있음 ✅ |
| 회원 | 동일 ✅ |

## 5. 문제점 — 전체 페이지 권한 점검

DB에서는 RLS가 쓰기를 막지만, **화면에서는 폼이 그대로 보이고 제출하면 조용히 실패**하는 페이지들이 있다.

### 5-1. 로그인 체크가 빠진 페이지 (LoginPrompt 추가 필요)

| 페이지 | 문제 | 위험도 |
|--------|------|--------|
| `/settings/withdraw` | 비회원도 계정 삭제 화면이 보임 | 심각 |
| `/settings/password` | 비회원도 비밀번호 변경 폼이 보임 | 심각 |
| `/settings/account` | 비회원도 계정 정보가 보임 | 심각 |
| `/settings` | 비회원도 설정 메인이 보임 | 심각 |
| `/settings/notifications` | 비회원도 알림 설정이 보임 | 심각 |
| `/profile/edit` | 비회원도 프로필 수정 폼이 보임 | 심각 |
| `/my/logs/write` | 비회원도 기록 작성 폼이 보임 | 높음 |
| `/my/logs/{id}/edit` | 비회원도 기록 수정 폼이 보임 | 높음 |
| `/my/selections` | 비회원도 셀렉션 목록이 보임 | 높음 |
| `/community/new` | 비회원도 게시글 작성 폼이 보임 | 높음 |
| `/used/write` | 비회원도 중고거래 글 작성 폼이 보임 | 높음 |

→ 이 11개 페이지에 `isLoggedIn` 체크 + LoginPrompt를 추가해야 함.

### 5-2. 읽기 페이지인데 쓰기 버튼에 로그인 체크가 없는 페이지

읽기 자체는 비회원도 가능하지만, 페이지 안의 **버튼/폼**이 로그인 없이 동작하려고 시도하는 경우.

| 페이지 | 문제가 되는 요소 | 해결 |
|--------|----------------|------|
| `/games/{id}` | "컬렉션 추가" 버튼, "평점" 버튼이 비회원한테도 보임 | 비회원 클릭 시 로그인 유도 시트(LoginPromptSheet) 띄우기 |
| `/community/{id}` | 좋아요 버튼, 댓글 입력창이 비회원한테도 보임 | 비회원 클릭 시 로그인 유도 |
| `/used/{id}` | 댓글 입력창이 비회원한테도 보임 | 비회원 클릭 시 로그인 유도 |
| `/my/selections/{id}` | 수정/삭제 메뉴(⋯ 버튼)가 본인이 아닌 사람에게도 보임 | 본인만 메뉴 표시 |

→ 이 4개 페이지는 **페이지 자체는 공개**하되, 쓰기 버튼에만 로그인 분기를 넣어야 함.

### 5-3. 소유자 확인이 빠진 페이지

로그인은 되어 있지만, **이 데이터가 내 것인지** 확인하지 않는 경우.

| 페이지 | 문제 | 해결 |
|--------|------|------|
| `/my/logs/{id}/edit` | 다른 사람의 기록 ID를 URL에 넣으면 수정 폼이 보임 | DB 조회 시 `user_id = 내 ID` 조건 추가, 아니면 404 |
| `/my/selections/{id}` | 다른 사람의 셀렉션인데 수정/삭제 메뉴가 보임 | 소유자만 메뉴 노출 |

→ RLS가 실제 수정은 막지만, **UI에서 미리 걸러야** 혼란이 없음.

### 5-4. 전체 페이지 현황 요약

```
✅ 로그인 체크 있음 (안전):
  ├── /my (컬렉션/기록/평점/셀렉션)
  ├── /my/games/[id] (게임별 기록)
  ├── /profile (내 프로필 보기)
  └── / (홈 — 최근기록 섹션만 분기)

✅ 공개 페이지 (체크 불필요):
  ├── /games, /games/[id] (게임 목록/상세)
  ├── /games/new, /games/ranking
  ├── /community (게시글 목록)
  ├── /local, /local/[id] (카페/매장)
  ├── /shop, /shop/[id] (상품)
  ├── /used (중고거래 목록)
  ├── /search (검색)
  ├── /u/[nickname] (공개 프로필)
  ├── /u/[nickname]/games/[id] (공개 게임 기록)
  ├── /login, /signup, /forgot-password, /reset-password
  └── /terms, /privacy, /guidelines

❌ 로그인 체크 빠짐 (수정 필요, 11개):
  ├── /settings (메인)
  ├── /settings/account
  ├── /settings/password
  ├── /settings/notifications
  ├── /settings/withdraw
  ├── /profile/edit
  ├── /my/logs/write
  ├── /my/logs/[id]/edit
  ├── /my/selections
  ├── /community/new
  └── /used/write

⚠️ 버튼에 로그인 분기 필요 (4개):
  ├── /games/[id] — 컬렉션/평점 버튼
  ├── /community/[id] — 좋아요/댓글
  ├── /used/[id] — 댓글
  └── /my/selections/[id] — 수정/삭제 메뉴 소유자 체크
```

## 6. 비공개 셀렉션 주의

`selections` 테이블만 다른 RLS 정책을 가지고 있다:

```
공개(is_public=true) → 누구나 볼 수 있음
비공개(is_public=false) → 본인만 볼 수 있음
```

공개 프로필(`/u/{닉네임}`)에서 셀렉션 탭을 추가할 때, 
비공개 셀렉션이 노출되지 않도록 주의해야 한다.
→ RLS가 이미 처리하지만, UI에서도 필터링하는 게 안전.

## 7. 결론 — 수정 작업 목록

### 1순위: 로그인 체크 추가 (11개 페이지)

각 페이지 상단에 아래 패턴 추가:
```
if (!isLoggedIn) → LoginPrompt 표시
```

대상: `/settings/*` 5개, `/profile/edit`, `/my/logs/write`, `/my/logs/[id]/edit`, `/my/selections`, `/community/new`, `/used/write`

### 2순위: 쓰기 버튼 로그인 분기 (4개 페이지)

비회원이 버튼 클릭 시 LoginPromptSheet 띄우기:
- `/games/[id]` — 컬렉션/평점 버튼
- `/community/[id]` — 좋아요/댓글
- `/used/[id]` — 댓글
- `/my/selections/[id]` — 수정/삭제 메뉴 소유자만 표시

### 3순위: RLS 실 테스트

- 비회원(로그아웃)으로 `/u/{닉네임}` 접속해서 컬렉션/통계 데이터 보이는지 확인
- 비회원으로 `/games/{id}`에서 컬렉션 추가 시도 → 실패하는지 확인
