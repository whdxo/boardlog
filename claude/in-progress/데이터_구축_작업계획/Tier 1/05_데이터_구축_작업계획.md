# 데이터 구축 작업 계획

> 보드게임 데이터를 DB에 넣고 서비스할 수 있는 상태로 만들기까지의 전체 작업 목록

---

## 0. 데이터 소스 현황 (2026년 기준 검증)

### CSV 데이터 소스

| 소스 | 상태 | 포함 데이터 | 최신성 |
|------|------|-----------|--------|
| **BGG 공식 덤프** (`bg_ranks`) | ✅ 매일 갱신 | ID, 이름, 순위, 평점, 평가수 | **최신** |
| **Kaggle (threnjen)** | ✅ 다운 가능 (154MB) | 2.2만 게임 × 47컬럼 + 태그/메카닉/디자이너 | ⚠️ 2022년 |
| **Kaggle (sujaykapadnis)** | ✅ 다운 가능 | 보드게임 데이터셋 | ⚠️ 2023년 |

### 현실적 한계

```
BGG 공식 덤프:
  ✅ 최신 데이터
  ❌ 기본 정보만 (이미지, 설명, 인원, 시간, 카테고리 없음)
  → 시딩 후 API /thing으로 상세 보강 필수

Kaggle 데이터셋:
  ✅ 상세 정보 풍부 (47개 컬럼)
  ❌ 2~3년 전 데이터 (최근 게임 없음)
  → 초기 시딩용은 OK, 최신 게임은 API로 보충

API만으로:
  ✅ 가장 최신
  ❌ rate limit (5초/1회) 때문에 대량 수집 느림
  → 2만개 전부는 비현실적, 인기 게임만 가능
```

### 추천 방식: 4단계 Tier 전략

```
Tier 1: CSV 전체 시딩 (137,000개) → 기본 정보
Tier 2: API 보강 (1,000개) → 이미지 + 상세
Tier 3: 수동 입력 (100개) → 한국어 + 가격
Tier 4: 실시간 수집 → 자동 확장
```

### CSV 실제 데이터 현황 (분석 완료)

## 1. 작업 전체 목록

### 자동 작업 (코드로 처리)

| # | 작업 | 설명 | 난이도 | 예상 시간 |
|---|------|------|--------|----------|
| A1 | BGG 공식 덤프 CSV 다운로드 | `bg_ranks` CSV (로그인 필요, 매일 갱신) | 쉬움 | 10분 |
| A2 | CSV → DB 시딩 스크립트 | Prisma seed — ID, 이름, 순위, 평점, 평가수 | 중간 | 3~4시간 |
| A3 | BGG API 연동 모듈 | bgg-xml-api-client + 인증 토큰 + 래퍼 함수 + 에러/재시도 | 중간 | 2~3시간 |
| A4 | API 상세 보강 스크립트 | 인기 1000개 → `/thing` 배치(20개씩) → 이미지/설명/인원/시간/태그 보강 | 중간 | 3~4시간 (코드) + 4~5시간 (실행, rate limit) |
| A5 | 한국어 이름 자동 추출 | BGG alternate name에서 한글 필터링 → title_kr 저장 | 쉬움 | 1시간 |
| A6 | 카테고리/메카닉 태그 파싱 | BGG link → tags + board_game_tags 테이블 매핑 | 중간 | 2~3시간 |
| A7 | 평점 10→5점 변환 | BGG 평점 ÷2 변환 로직 | 쉬움 | 30분 |
| A8 | 핫 리스트 cron | `/hot` 50개 일일 자동 갱신 API | 중간 | 2시간 |
| A9 | 검색 시 실시간 수집 | 유저 검색 → DB에 없으면 BGG에서 수집 → 저장 | 중간 | 3~4시간 |
| A10 | 이미지 URL 검증 | BGG 이미지 링크 깨진 거 체크 스크립트 | 쉬움 | 1시간 |

> ⚠️ A4 실행 시 rate limit 주의: 1000개 ÷ 20(배치) × 5초 = 최소 250초(~4분). 실제로는 네트워크 지연 포함 30분~1시간. 에러/재시도 포함하면 수 시간.

### 수동 작업 (사람이 직접)

| # | 작업 | 설명 | 난이도 | 예상 시간 |
|---|------|------|--------|----------|
| M1 | 인기 게임 100개 한국어 이름 | BGG에 없는 한국어 이름 직접 입력 | 단순 노동 | 2~3시간 |
| M2 | 인기 게임 100개 가격 입력 | 네이버 쇼핑에서 검색 → 가격 복사 | 단순 노동 | 3~4시간 |
| M3 | 인기 게임 100개 구매 링크 | 네이버 쇼핑 URL 복사 → DB 입력 | 단순 노동 | M2와 동시 |
| M4 | 국내 전용 게임 20개 등록 | BGG에 없는 게임 전체 정보 직접 입력 | 중간 | 2~3시간 |
| M5 | 카테고리 한국어 번역 | Strategy→전략, Family→가족 등 태그 번역 (~30개) | 쉬움 | 1시간 |
| M6 | 홈 배너 3장 제작 | 이미지 디자인 + 업로드 | 중간 | 1~2시간 |
| M7 | 추천 게임 8개 선정 | 에디터 픽 큐레이션 | 쉬움 | 30분 |
| M8 | 한국어 설명 번역 (인기 30개) | 영문 description → 한국어 (AI 번역 + 검수) | 중간 | 3~4시간 |

## 2. 복잡도 분석

### 쉬운 것 (단순 작업)

```
✅ BGG 공식 덤프 CSV 다운로드 → 로그인 후 클릭
✅ 평점 변환 (÷2) → 함수 1줄
✅ 한국어 이름 자동 추출 → 정규식 한글 필터
✅ 이미지 URL은 BGG CDN 그대로 사용 (다운로드/리사이즈 불필요)
✅ 카테고리 한국어 번역 → 30개 정도 매핑 테이블
```

### 보통인 것 (코드 작성 필요하지만 어렵지 않음)

```
🟡 CSV 시딩 스크립트 → Prisma upsert 반복문, 배치 처리
🟡 BGG API 래퍼 → npm 패키지 설치 + 토큰 세팅 + 에러 핸들링
🟡 태그 파싱 → link 타입별 분류 + M:N 관계 저장
🟡 핫 리스트 cron → Vercel cron + API 라우트
🟡 수동 데이터 입력 → 반복적이지만 단순
```

### 까다로운 것 (신경 써야 하는 부분)

```
🔴 BGG API rate limit 관리
   → 5초 간격 대기, 배치 요청 20개씩
   → 1000개 보강에 수 시간 소요 (스크립트 돌려놓고 대기)

🔴 BGG 공식 덤프 CSV 구조
   → 기본 정보만 있어서 API 보강 필수
   → 덤프 컬럼: id, name, yearpublished, rank, bayesaverage, average, usersrated
   → 없는 것: 이미지, 설명, 인원, 시간, 카테고리, 메카닉, 디자이너

🔴 검색 시 실시간 수집
   → 유저 응답 속도 vs BGG API 지연 (5초+) 균형
   → 백그라운드 수집 + 캐시 전략 필요

🔴 description HTML 디코딩
   → &amp; &#10; 등 엔티티 변환 + 불필요한 태그 제거

🔴 한국어 이름 커버리지
   → BGG에서 ~5%만 자동, 나머지 수동
   → 지속적 보강 필요

🔴 Kaggle 데이터 사용 시 주의
   → 2~3년 전 데이터 → 최근 게임/평점 부정확
   → 공식 덤프 CSV와 병합 시 충돌 관리 필요
```

## 3. 전체 일정 (Tier별)

### Tier 1 — CSV 전체 시딩 (2일)

```
Day 1:
  ├── Prisma 스키마 확정 + 마이그레이션
  ├── csv-parse, tsx 패키지 설치
  ├── CSV 시딩 스크립트 작성
  └── 실행: 137,000개 기본 게임 일괄 insert (~3~5분)
        ├── bgg_id, title, year_published, bgg_rank
        ├── avg_rating (÷2 변환), rating_count
        ├── is_expansion 필터 (확장팩 제외)
        └── 카테고리 순위 컬럼 → 태그 자동 연결 (전략/가족/파티 등 8종)

Day 2:
  ├── 시딩 결과 검증 (npx prisma studio)
  ├── 평점 변환 정확성 체크
  ├── 카테고리 태그 한국어 번역 매핑 (8개)
  └── 데이터 정합성 테스트

결과: DB에 137,000개 게임 기본 정보 + 8종 카테고리 태그
단, 이미지/설명/인원/시간 없음 → Tier 2에서 보강
```

### Tier 2 — API 상세 보강 (3~4일)

> ⚠️ BGG API 토큰 승인 필요. 토큰 대기 중에는 Tier 1 + 프론트 개발 진행.

```
Day 3:
  ├── BGG API 연동 모듈 작성
  │     ├── xml2js 파싱
  │     ├── 인증 토큰 헤더
  │     ├── rate limit (5초 대기)
  │     ├── 배치 요청 (20개씩)
  │     └── 에러/재시도 로직
  ├── API 상세 보강 스크립트 작성
  └── 인기 상위 500개 보강 실행 시작
        ├── 이미지 (thumbnail + image)
        ├── 설명 (description, HTML 디코딩)
        ├── 인원 (min~max players)
        ├── 시간 (min~max playtime)
        ├── 연령 (minage)
        ├── 디자이너, 퍼블리셔
        ├── 상세 카테고리 (boardgamecategory)
        ├── 메카닉 (boardgamemechanic)
        ├── 한국어 이름 자동 추출 (alternate name 중 한글)
        ├── 난이도 (averageweight)
        └── last_synced_at 업데이트

Day 4:
  ├── 501~1000위 보강 실행
  ├── 카테고리/메카닉 → tags 테이블 매핑 정리
  └── 이미지 URL 검증 (깨진 링크 체크)

Day 5~6 (여유분):
  ├── 에러 케이스 재시도
  ├── 데이터 검수 (이미지 있는지, 설명 있는지)
  └── 필요시 추가 보강 (1000~3000위)

결과: 인기 1,000개 게임에 이미지+설명+인원+시간+태그+한국어명(~50개)
API 호출: 1000 ÷ 20 = 50회 × 5초 = ~4분 (최소), 실제 30분~1시간
```

### Tier 3 — 수동 입력 (3~4일)

```
Day 7:
  ├── 인기 게임 100개 한국어 이름 입력
  │     (data/korean_names.json 작성 → 스크립트 실행)
  └── 인기 게임 100개 가격 + 구매 링크 입력
        (네이버 쇼핑 검색 → data/purchase_links.json → 스크립트)

Day 8:
  ├── 국내 전용 게임 20개 등록 (BGG에 없는 게임)
  └── 인기 30개 한국어 설명 번역 (AI 번역 + 수동 검수)

Day 9:
  ├── 홈 배너 3장 제작 + Supabase Storage 업로드
  ├── 추천 게임 8개 선정
  └── 전체 데이터 검수 (이미지 깨짐, 누락, 오류)

Day 10 (여유분):
  ├── 누락/오류 보강
  └── 수동 입력 마무리

결과: 인기 100개 한국어+가격+구매링크 + 국내 게임 20개 + 배너
```

### Tier 4 — 자동화 (3~4일)

```
Day 11:
  ├── 핫 리스트 cron API (Vercel cron, 매일 06:00)
  │     BGG /hot → 50개 → DB 갱신
  └── 검색 시 실시간 수집 로직
        유저 검색 → DB에 없으면 → BGG /search + /thing → 저장

Day 12:
  ├── 데이터 신선도 관리 (last_synced_at 기반)
  │     7일 초과 → 유저 방문 시 백그라운드 갱신
  ├── 주간 평점 갱신 배치 스크립트
  └── vercel.json cron 설정

Day 13:
  ├── 전체 파이프라인 테스트
  ├── 에러 핸들링 보강
  └── 최종 검수 + 완료

결과: 핫 50 자동 갱신 + 검색 시 자동 확장 + 주간 평점 갱신
```

### 런칭 최소 조건 (Tier 1 + 2 + 3)

```
DB에 137,000개 기본정보 + 인기 1,000개 상세(이미지/설명)
+ 100개 한국어/가격/구매링크 + 배너 3장
→ 약 8~10일이면 서비스 가능한 상태
```

### 단계별 서비스 가능 수준

```
Tier 1만 완료 (2일):
  → DB에 게임 있지만 이미지 없어서 카드 UI 안 됨
  → 프론트 개발용 더미 대체로는 사용 가능

Tier 1+2 완료 (5~6일):
  → 인기 1,000개 게임은 완전한 상세 페이지 가능
  → 영문 전용, 가격 없음

Tier 1+2+3 완료 (8~10일):
  → 한국어 서비스 가능 (인기 100개)
  → 구매 링크 있음 → 핵심 흐름 완성

Tier 4 완료 (11~14일):
  → 자동 확장 + 갱신 → 운영 준비 완료
```

### 없어도 런칭은 가능한 것

```
- Tier 4 자동화 → 런칭 후 추가 가능
- 한국어 설명 번역 → 영문으로 먼저 런칭, 점진 번역
- 1,000개 넘는 게임 상세 보강 → 유저 검색 시 실시간 수집
- 국내 전용 게임 → 런칭 후 수시 추가 가능
```
