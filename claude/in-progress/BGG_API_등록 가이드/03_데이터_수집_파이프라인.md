# 데이터 수집 파이프라인

> BGG API + Kaggle 데이터를 활용한 보드게임 데이터 수집 전략

---

## 1. 전체 파이프라인 개요

```
┌─────────────────────────────────────────────────────────┐
│                    데이터 수집 파이프라인                   │
│                                                         │
│  [1단계] Kaggle CSV ──→ DB 시딩 (2만+ 게임 일괄)          │
│              ↓                                          │
│  [2단계] BGG /thing ──→ 상세 보강 (이미지, 평점, 태그)     │
│              ↓                                          │
│  [3단계] 운영 중 ──→ 유저 검색 시 실시간 수집              │
│              ↓                                          │
│  [4단계] 배치 ──→ 일일/주간 자동 갱신                     │
│              ↓                                          │
│  [5단계] 수동 ──→ 한국어 이름, 가격, 국내 게임 입력         │
└─────────────────────────────────────────────────────────┘
```

### 시딩 스크립트

```typescript
// scripts/seed-from-kaggle.ts

import { parse } from 'csv-parse/sync';
import { readFileSync } from 'fs';
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();
const csv = readFileSync('data/games.csv', 'utf-8');
const games = parse(csv, { columns: true, skip_empty_lines: true });

async function seed() {
  console.log(`시딩 시작: ${games.length}개 게임`);

  // 배치 처리 (500개씩)
  for (let i = 0; i < games.length; i += 500) {
    const batch = games.slice(i, i + 500);

    await prisma.$transaction(
      batch.map((game: any) =>
        prisma.boardGame.upsert({
          where: { bgg_id: parseInt(game.BGGId) },
          update: {
            avg_rating: parseFloat(game.AvgRating) / 2 || 0,
            rating_count: parseInt(game.NumUserRatings) || 0,
          },
          create: {
            bgg_id: parseInt(game.BGGId),
            title: game.Name || '',
            description: decodeHtml(game.Description || ''),
            min_players: parseInt(game.MinPlayers) || 0,
            max_players: parseInt(game.MaxPlayers) || 0,
            play_time_min: parseInt(game.ComMinPlaytime) || parseInt(game.MfgPlaytime) || 0,
            play_time_max: parseInt(game.ComMaxPlaytime) || parseInt(game.MfgPlaytime) || 0,
            min_age: parseInt(game.ComAgeRec) || parseInt(game.MfgAgeRec) || 0,
            year_published: parseInt(game.YearPublished) || 0,
            image_url: game.ImagePath || '',
            designer: '',   // 별도 CSV에서 매핑
            publisher: '',  // 별도 CSV에서 매핑
            avg_rating: parseFloat(game.AvgRating) / 2 || 0,
            rating_count: parseInt(game.NumUserRatings) || 0,
          },
        })
      )
    );

    console.log(`${i + batch.length}/${games.length} 완료`);
  }
}

function decodeHtml(html: string): string {
  return html
    .replace(/&amp;/g, '&')
    .replace(/&lt;/g, '<')
    .replace(/&gt;/g, '>')
    .replace(/&quot;/g, '"')
    .replace(/&#10;/g, '\n');
}

seed();
```

## 3. 2단계 — BGG API 상세 보강

Kaggle 데이터에 없거나 오래된 정보를 BGG API로 보강.

### 보강 대상

| 데이터 | 이유 |
|--------|------|
| 고해상도 이미지 | Kaggle 이미지 URL이 깨진 경우 |
| 최신 평점 | Kaggle 데이터는 스냅샷 (오래됨) |
| 한국어 이름 | Kaggle에 없음 → BGG alternate name에서 추출 |
| 카테고리/메카닉 | tags 테이블에 정확한 매핑 |

### 보강 스크립트

```typescript
// scripts/enrich-from-bgg.ts

import { BggClient } from 'bgg-xml-api-client';
import { PrismaClient } from '@prisma/client';

const bgg = BggClient.create();
const prisma = new PrismaClient();

async function enrich() {
  // bgg_id가 있고 last_synced_at이 null인 게임들
  const games = await prisma.boardGame.findMany({
    where: { bgg_id: { not: null }, last_synced_at: null },
    take: 100, // 한 번에 100개씩
    orderBy: { rating_count: 'desc' }, // 인기 게임 우선
  });

  // 20개씩 배치 호출
  for (let i = 0; i < games.length; i += 20) {
    const batch = games.slice(i, i + 20);
    const ids = batch.map(g => g.bgg_id!);

    const result = await bgg.thing({
      id: ids,
      stats: 1,
      type: 'boardgame',
    });

    for (const item of result.items) {
      const koreanName = item.names?.find(
        (n: any) => n.type === 'alternate' && /[가-힣]/.test(n.value)
      );

      await prisma.boardGame.update({
        where: { bgg_id: item.id },
        data: {
          image_url: item.image || undefined,
          thumbnail_url: item.thumbnail || undefined,
          title_kr: koreanName?.value || undefined,
          avg_rating: item.statistics?.ratings?.average
            ? parseFloat(item.statistics.ratings.average) / 2
            : undefined,
          rating_count: item.statistics?.ratings?.usersRated
            ? parseInt(item.statistics.ratings.usersRated)
            : undefined,
          weight: item.statistics?.ratings?.averageWeight
            ? parseFloat(item.statistics.ratings.averageWeight)
            : undefined,
          bgg_rank: item.statistics?.ratings?.ranks?.[0]?.value
            ? parseInt(item.statistics.ratings.ranks[0].value)
            : undefined,
          last_synced_at: new Date(),
        },
      });
    }

    // Rate limit 준수 (5초 대기)
    await sleep(5000);
    console.log(`${i + batch.length}/${games.length} 보강 완료`);
  }
}

function sleep(ms: number) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

enrich();
```

## 4. 3단계 — 유저 검색 시 실시간 수집

```typescript
// src/lib/bgg-sync.ts

export async function searchAndSync(query: string) {
  // 1. 우리 DB에서 먼저 검색
  const localResults = await prisma.boardGame.findMany({
    where: {
      OR: [
        { title: { contains: query, mode: 'insensitive' } },
        { title_kr: { contains: query, mode: 'insensitive' } },
      ],
    },
    take: 20,
  });

  // 2. 결과가 5개 미만이면 BGG에서 추가 검색
  if (localResults.length < 5) {
    const bggResults = await bgg.search({
      query,
      type: 'boardgame',
    });

    // 3. 우리 DB에 없는 게임만 필터
    const newIds = bggResults.items
      .map((item: any) => item.id)
      .filter((id: number) => !localResults.some(g => g.bgg_id === id))
      .slice(0, 20); // 최대 20개

    // 4. BGG에서 상세 수집 → DB 저장
    if (newIds.length > 0) {
      const details = await bgg.thing({
        id: newIds,
        stats: 1,
        type: 'boardgame',
      });

      for (const item of details.items) {
        await saveGameFromBgg(item); // upsert 로직
      }
    }
  }

  // 5. 최종 DB 검색 결과 반환
  return prisma.boardGame.findMany({
    where: {
      OR: [
        { title: { contains: query, mode: 'insensitive' } },
        { title_kr: { contains: query, mode: 'insensitive' } },
      ],
    },
    take: 20,
    orderBy: { rating_count: 'desc' },
  });
}
```

## 5. 4단계 — 자동 배치 갱신

### cron 스케줄

| 작업 | 주기 | 내용 |
|------|------|------|
| 핫 리스트 갱신 | 매일 06:00 | BGG `/hot` → 50개 캐시 갱신 |
| 인기 게임 평점 갱신 | 매주 월 03:00 | rating_count 상위 500개 → `/thing` 배치 |
| 이미지 깨짐 체크 | 매주 일 03:00 | image_url HTTP HEAD 체크 → 깨진 거 재수집 |

### Vercel Cron 설정

```json
// vercel.json
{
  "crons": [
    {
      "path": "/api/cron/sync-hot",
      "schedule": "0 6 * * *"
    },
    {
      "path": "/api/cron/sync-ratings",
      "schedule": "0 3 * * 1"
    }
  ]
}
```

### 핫 리스트 갱신 API

```typescript
// src/app/api/cron/sync-hot/route.ts

export async function GET(request: Request) {
  // Vercel Cron 인증 체크
  if (request.headers.get('Authorization') !== `Bearer ${process.env.CRON_SECRET}`) {
    return Response.json({ error: 'Unauthorized' }, { status: 401 });
  }

  const hot = await bgg.hot({ type: 'boardgame' });

  for (const item of hot.items) {
    // DB에 없으면 상세 수집 후 저장
    const exists = await prisma.boardGame.findUnique({
      where: { bgg_id: item.id },
    });

    if (!exists) {
      const detail = await bgg.thing({ id: item.id, stats: 1 });
      await saveGameFromBgg(detail.items[0]);
      await sleep(5000); // rate limit
    }
  }

  // 핫 리스트 캐시 갱신
  await prisma.hotGame.deleteMany();
  await prisma.hotGame.createMany({
    data: hot.items.map((item: any, index: number) => ({
      bgg_id: item.id,
      rank: index + 1,
    })),
  });

  return Response.json({ success: true, count: hot.items.length });
}
```

## 7. 데이터 신선도 관리

```
┌──────────────────────────────────────────┐
│             last_synced_at 기준            │
│                                          │
│  NULL          → 아직 BGG 보강 안 됨       │
│  7일 이내       → 최신 (갱신 불필요)        │

│  유저가 상세 페이지 방문 시:                 │
│  last_synced_at이 7일 초과면               │
│  → 백그라운드에서 BGG 재호출 + 갱신         │
```

## 10. 수동 데이터 입력 파이프라인

BGG에 없는 데이터를 효율적으로 입력하는 작업 흐름.

### 10.1 한국어 게임명 입력 흐름

```
1단계: BGG alternate name에서 자동 추출
  └── 한글(가-힣) 포함된 alternate name 자동 저장
  └── 약 5%만 커버됨

2단계: 인기 게임 수동 매핑 (런칭 전)
  └── BGG rank 상위 100개 게임
  └── 관리자가 한국어 이름 직접 입력
  └── 소요: 약 2~3시간

3단계: 지속 확장 (운영 중)
  └── 유저가 검색한 게임 중 한국어명 없는 것 리포트
  └── 주 1회 관리자 보강
  └── Phase 2: 유저 제보 기능
```

### 10.2 가격 + 구매 링크 입력 흐름

```
1단계: 인기 게임 100개 (런칭 전)
  │
  ├── 네이버 쇼핑에서 게임명 검색
  ├── 최저가 + 쇼핑몰 URL 복사
  └── purchase_links 테이블에 입력
      ├── shop_name: "네이버 쇼핑"
      ├── url: 검색 결과 URL
      ├── price: 최저가 (원)
      └── is_available: true

2단계: 점진 확대 (운영 중)
  └── 주 1회 인기 게임 20~30개씩 추가
  └── 가격 변동 시 수동 업데이트

3단계: 자동화 (Phase 2)
  └── 네이버 쇼핑 API or 크롤링
  └── 가격 자동 갱신
  └── 품절 자동 체크
```

### 10.3 국내 전용 게임 등록 흐름

BGG에 없는 한국 보드게임 (예: 할리갈리 한국판, 블루마블 등)

```
관리자가 직접 모든 필드 입력:
  ├── title: 게임명 (한국어)
  ├── title_kr: 동일
  ├── description: 게임 소개 직접 작성
  ├── image_url: 직접 촬영/제작 → Supabase Storage 업로드
  ├── min_players ~ min_age: 박스 정보 참고
  ├── designer, publisher: 직접 입력
  ├── price: 검색 후 입력
  ├── purchase_links: 쇼핑몰 URL
  ├── bgg_id: null (BGG에 없으므로)
  └── tags: 직접 분류
```

### 10.4 홈 배너 관리 흐름

```
관리자 작업:
  ├── 배너 이미지 제작 (1200×400px, 16:9 모바일용도 별도)
  ├── Supabase Storage > banners 버킷에 업로드
  └── banners 테이블에 등록:
      ├── image_url: 업로드된 URL
      ├── title: 배너 제목
      ├── subtitle: 부제
      ├── link_url: 클릭 시 이동 URL (게임 상세 or 외부)
      ├── is_active: true/false
      └── order: 표시 순서
```

### 10.5 수동 작업 예상 시간

| 작업 | 대상 | 소요 시간 | 빈도 |
|------|------|----------|------|
| 한국어 이름 100개 | 인기 게임 | 2~3시간 | 런칭 전 1회 |
| 가격 + 구매 링크 100개 | 인기 게임 | 3~4시간 | 런칭 전 1회 |
| 국내 게임 20개 등록 | 유명 국내 게임 | 2~3시간 | 런칭 전 1회 |
| 배너 3장 제작 | 홈 배너 | 1~2시간 | 런칭 전 1회 |
| 추천 게임 선정 | 8개 | 30분 | 주 1회 |
| 가격 업데이트 | 20~30개 | 1시간 | 주 1회 |
| 한국어 이름 추가 | 20~30개 | 30분 | 주 1회 |
| **런칭 전 총 소요** | | **약 10~12시간** | |
