# 백엔드 구현 로드맵

> 원칙: **커뮤니티(회원 간 소통)를 먼저**, 그 다음 개인 기록, 마지막으로 부가 기능

---

## Phase 1 — 커뮤니티 핵심 (가장 먼저)

> 비회원도 읽고, 회원이 쓰고, 회원끼리 반응하는 구조

### 1-1. 커뮤니티 게시글 CRUD

**목표**: 목업 데이터 → 실제 DB 연동

| 작업 | 설명 | 파일 위치 |
|------|------|-----------|
| 게시글 목록 API | 최신순/추천순, 카테고리 필터, 페이지네이션 | `app/api/posts/route.ts` |
| 게시글 상세 API | 조회수 증가 포함 | `app/api/posts/[id]/route.ts` |
| 게시글 생성 | 로그인 필수, 게임 태그 연결 | `app/api/posts/route.ts` POST |
| 게시글 수정/삭제 | 본인만 가능 (RLS) | `app/api/posts/[id]/route.ts` |
| 커뮤니티 페이지 연동 | 목업 → React Query 훅 | `app/(main)/community/page.tsx` |
| 게시글 상세 페이지 연동 | 내용 + 조회수 | `app/(main)/community/[id]/page.tsx` |
| 글쓰기 폼 연동 | 실제 저장 버튼 작동 | `app/(main)/community/new/page.tsx` |

**팔로우 피드**: 팔로우 기능이 구현된 후 Phase 2에서 연동.

---

### 1-2. 댓글 시스템

| 작업 | 설명 |
|------|------|
| 댓글 목록 조회 | 게시글 ID로 댓글 + 대댓글 트리 |

### 1-3. 좋아요 & 북마크

| 작업 | 설명 |
|------|------|
| 게시글 좋아요 토글 | post_likes 테이블, 비로그인 시 로그인 유도 |
| 좋아요 수 캐시 갱신 | posts.like_count 업데이트 |

## Phase 2 — 소셜 기능 (회원 간 연결)

### 2-1. 팔로우 / 팔로워

| 작업 | 설명 |
|------|------|
| 팔로우/언팔로우 API | follows 테이블 INSERT/DELETE |

### 2-2. 알림 시스템

| 알림 종류 | 트리거 시점 |
|-----------|------------|
| `comment` | 내 게시글에 댓글 달릴 때 |
| `like` | 내 게시글에 좋아요 받을 때 |
| `follow` | 누군가 나를 팔로우할 때 |

| 작업 | 설명 |
|------|------|
| 알림 생성 로직 | 댓글/좋아요/팔로우 API에서 notifications 테이블에 INSERT |
| 알림 목록 API | 본인 알림만, 읽음/안읽음 구분 |
| 알림 읽음 처리 | PATCH /api/notifications/[id] |
| 알림 페이지 연동 | `/notifications` 페이지 실데이터 |
| 실시간 배지 | GNB 알림 아이콘 (폴링 or Supabase Realtime) |

## 구현 시 공통 패턴

```
// Route Handler 표준 패턴
export async function POST(req: Request) {
  const supabase = await createSupabaseServerClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return Response.json({ error: 'Unauthorized' }, { status: 401 })
  
  // ... 비즈니스 로직
}
```

- 모든 mutation은 서버 Route Handler에서 처리
- 인증 확인 → 입력 검증 → DB 작업 → 응답
- RLS가 최후 방어선 (API에서도 user_id 검증)
