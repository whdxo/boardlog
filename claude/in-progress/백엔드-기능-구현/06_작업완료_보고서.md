### Phase 1 — 커뮤니티

**Route Handlers (신규 5개)**

| 엔드포인트 | 메서드 | 핵심 동작 |
|-----------|--------|-----------|
| `/api/posts` | GET | 카테고리·피드(latest/best/following) 필터, isLiked 주입 |
| `/api/posts` | POST | 제목·내용 유효성 검사 후 INSERT |
| `/api/posts/[id]` | GET | 상세 조회 + view_count +1 (비동기, non-blocking) |
| `/api/posts/[id]` | PATCH/DELETE | 소유권 검증(403) 후 수정/삭제 |
| `/api/posts/[id]/like` | POST | maybeSingle로 상태 확인 → 토글 + like_count ±1 |
| `/api/posts/[id]/comments` | GET/POST | 조회 시 buildCommentTree 적용, 대댓글 깊이 1단 제한 |
| `/api/comments/[id]` | DELETE | 소유권 검증, **대댓글 포함 comment_count 차감** |

**훅 (신규 4개)**: `usePosts` · `usePost` · `useComments` · `usePostMutations`

**페이지 연동**: community/page, community/[id]/page, community/new/page — Mock 전부 제거

### Phase 3 — 개인 기록

**DB 마이그레이션**

```sql
-- supabase/migrations/00002_play_logs_visibility.sql
ALTER TABLE play_logs ADD COLUMN IF NOT EXISTS visibility TEXT NOT NULL DEFAULT 'public'
  CHECK (visibility IN ('public', 'private'));
-- play_logs_select RLS 정책 재작성: public OR auth.uid() = user_id
```

**Route Handlers (신규 4개)**

| 엔드포인트 | 메서드 | 핵심 동작 |
|-----------|--------|-----------|
| `/api/collections` | POST/DELETE | UPSERT (user_id+game_id 유니크), DELETE with gameId query param |
| `/api/ratings` | POST/DELETE | UPSERT + avg_rating · rating_count 재계산 |
| `/api/play-logs` | POST | INSERT + play_log_scores 삽입 |
| `/api/play-logs/[id]` | GET/PATCH/DELETE | private 접근 403, scores 재동기화 |

**훅 (신규 3개)**: `useCollection` · `useRating` · `usePlayLog`  
— 낙관적 업데이트 (collectionStore) + 실패 시 롤백

**페이지/컴포넌트 연동**
- `LogForm.tsx` — visibility 토글 UI 추가
- my/logs/write, my/logs/[id], my/logs/[id]/edit — Mock 제거 + 실데이터
- games/[id]/page — CollectionSheet·RatingModal → 실제 mutation 연동

**스토어 초기화**  
`authStore.ts` 로그인 시 collections + ratings 백그라운드 fetch → `collectionStore.initFromServer()`  
로그아웃 시 `collectionStore.reset()`

## 2. 검토 후 수정 사항

### Critical

| 항목 | 파일 | 수정 내용 |
|------|------|-----------|
| 댓글 삭제 시 대댓글 카운트 누락 | `api/comments/[id]/route.ts` | 삭제 전 대댓글 수 조회, `1 + replyCount`만큼 차감 |
| 팔로우 상태 캐시 미갱신 | `hooks/useFollow.ts` | queryKey에 `isLoggedIn` 추가, setQueryData 키 통일 |

### Warning

| 항목 | 파일 | 수정 내용 |
|------|------|-----------|
| `Post.user` non-nullable 타입 불일치 | `types/index.ts` | `user: User \| null`로 변경 |
| `post.user` 직접 접근 런타임 오류 가능 | `PostCard.tsx`, `community/[id]/page.tsx` | 옵셔널 체이닝 + 기본값 처리 |

## 4. 배포 전 필수 작업

1. **Supabase 마이그레이션 실행**
   ```bash
   supabase db push
   # 또는 Dashboard SQL Editor에서 00002_play_logs_visibility.sql 실행
   ```

2. **RLS 정책 확인**  
   - `play_logs`: visibility=public OR auth.uid()=user_id
   - `collections`, `ratings`: user_id=auth.uid() (INSERT/UPDATE/DELETE)
   - `posts`, `comments`: 읽기 공개, 쓰기 auth.uid() 필요

3. **Storage 버킷 생성 확인**
   - `avatars` 버킷 — public read, authenticated write (upsert)
   - `post-images` 버킷 — 이미지 업로드 구현 시 필요

4. **환경 변수 확인**  
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
