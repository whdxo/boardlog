## Q5 상세: 공개/비공개 권한 설계

"여러 권한 방향에서 자세하게 구현" 요청에 따른 설계:

### 대상 데이터
| 데이터 | 현재 RLS | 변경 방향 |
|--------|----------|-----------|
| `play_logs` | 누구나 읽기 | 개별 `is_public` 컬럼 추가 |
| `collections` | 누구나 읽기 | 프로필 공개 설정에 연동 |
| `profiles` | 누구나 읽기 | `is_private` 계정 설정 추가 |

### 권한 레벨 (3단계)
```
1. 전체 공개 (기본값) — 비회원 포함 누구나 볼 수 있음
2. 팔로워만 — 나를 팔로우한 사람만 볼 수 있음 (Phase 2 이후)
3. 나만 보기 — 본인만
```

### 구현 방향

**play_logs 테이블에 컬럼 추가 필요 (마이그레이션)**:
```sql
ALTER TABLE play_logs ADD COLUMN visibility TEXT NOT NULL DEFAULT 'public'
  CHECK (visibility IN ('public', 'private'));
```

**RLS 정책 변경 필요**:
```sql
-- 기존: USING (true)
-- 변경: 본인 로그이거나, public인 경우만
DROP POLICY "play_logs_select" ON play_logs;
CREATE POLICY "play_logs_select" ON play_logs FOR SELECT
  USING (user_id = auth.uid() OR visibility = 'public');
```

**collections 공개 여부**:
컬렉션은 프로필 수준에서 통제 (profiles 테이블에 `collection_visibility` 추가 or 심플하게 기본 공개 유지).  
→ 초기에는 컬렉션은 항상 공개, 플레이 로그만 개별 공개/비공개 지원.

### UI 반영 위치
- 로그 작성 폼(`LogForm`): "공개 범위" 선택 (공개/비공개)
- 로그 상세/수정 페이지: 공개 범위 변경 가능
- 설정 페이지: 기본 공개 범위 설정
