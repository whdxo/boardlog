## 4. 기술 방향

**쓰기 작업 구현 방식**:  
Supabase 클라이언트(server-side)를 써서 Next.js Route Handler(`app/api/...`)에 구현.  
이유: 서버에서 인증 확인 → 안전, Supabase RLS 활용 → 정책 강제 적용.

**이미지 업로드**:  
Supabase Storage API → 버킷별 정책 이미 설정됨.  
클라이언트에서 직접 업로드 (인증된 사용자만 가능, RLS로 제어).

**알림**:  
DB 트리거 or 서버 액션에서 직접 `notifications` 테이블에 INSERT.  
실시간 표시: Supabase Realtime 구독 or 폴링.
